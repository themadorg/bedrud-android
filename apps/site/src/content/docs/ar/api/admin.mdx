نقاط نهاية المشرفين تحت `/api/admin/`. جميعها تتطلب مستوى وصول **superadmin** ورمز JWT صالح في الرأس `Authorization: Bearer <token>`.

## نظرة عامة

| نقطة النهاية | الطريقة | الوصف |
|---|---|---|
| `/api/admin/users` | GET | عرض جميع المستخدمين (مقسمة لصفحات) |
| `/api/admin/users/:id` | GET | عرض تفاصيل مستخدم مع غرفه |
| `/api/admin/users/:id/status` | PUT | تفعيل أو تعطيل مستخدم |
| `/api/admin/users/:id/accesses` | PUT | تحديث صلاحيات مستخدم |
| `/api/admin/users/:id` | DELETE | حذف مستخدم نهائياً |
| `/api/admin/rooms` | GET | عرض جميع الغرف (مقسمة لصفحات) |
| `/api/admin/rooms/:roomId` | PUT | تحديث إعدادات أي غرفة |
| `/api/admin/rooms/:roomId` | DELETE | حذف غرفة (مهمة خلفية) |
| `/api/admin/rooms/:roomId/suspend` | POST | تعليق غرفة (مهمة خلفية) |
| `/api/admin/rooms/bulk/suspend` | POST | تعليق جماعي للغرف |
| `/api/admin/rooms/bulk/close` | POST | إغلاق جماعي للغرف |
| `/api/admin/users/bulk/delete` | POST | حذف جماعي للمستخدمين |
| `/api/admin/rooms/:roomId/participants` | GET | جلب بيانات المشاركين المباشرين من LiveKit |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | طرد أي مشارك من غرفة |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | كتم صوت أي مشارك |
| `/api/admin/settings` | GET | عرض إعدادات النظام (الأسرار مُخفّاة) |
| `/api/admin/settings` | PUT | تحديث إعدادات النظام |
| `/api/admin/invite-tokens` | GET | عرض جميع رموز الدعوة |
| `/api/admin/invite-tokens` | POST | إنشاء رمز دعوة جديد |
| `/api/admin/invite-tokens/:id` | DELETE | حذف رمز دعوة |
| `/api/admin/online-count` | GET | إجمالي المشاركين النشطين |
| `/api/admin/livekit/stats` | GET | إحصائيات خادم LiveKit المجمّعة |
| `/api/admin/cert-info` | GET | بيانات شهادة TLS |

## مستويات الوصول

يستخدم Bedrud نظام وصول متعدد المستويات. كل مستخدم لديه مصفوفة `accesses` يمكنها احتواء مستويات متعددة في آنٍ واحد.

| مستوى الوصول | الوصف | حماية الوسيط |
|---|---|---|
| `superadmin` | وصول كامل للنظام. لوحة المشرف، إدارة المستخدمين والغرف، الإعدادات، رموز الدعوة. | `RequireAccess("superadmin")` على `/api/admin/*` |
| `admin` | معرّف لكن غير مفروض بواسطة وسيط. معلوماتي فقط. | لا يوجد حالياً |
| `moderator` | إشراف على مستوى الغرفة. حظر الدردشة، كتم/إلغاء كتم المشاركين، تسليط الضوء، تعطيل الفيديو لكل غرفة. | فحص `isRoomModerator()` بنطاق الغرفة |
| `user` | مستخدم مُصادق عادي. إنشاء/الانضمام للغرف، إدارة الملف الشخصي. | `Protected()` على مسارات المستخدم |
| `guest` | مستخدم مؤقت للانضمام للغرف العامة. تسجيل دخول الضيوف فقط. | غير متاح |

> **ملاحظة:** مجموعة مسارات `/api/admin/*` تتحقق من `superadmin`، وليس `admin`. فقط المستخدمون الذين لديهم `superadmin` في مصفوفة `accesses` يمكنهم استخدام نقاط النهاية هذه.

## إدارة المستخدمين

### عرض جميع المستخدمين

استرجاع قائمة مقسمة لصفحات بجميع المستخدمين المُسجَّلين.

```
GET /api/admin/users?page=1&limit=50
```

**الرؤوس:** `Authorization: Bearer <accessToken>`

**معاملات الاستعلام:**

| المعامل | النوع | الافتراضي | الوصف |
|---|---|---|---|
| `page` | int | 1 | رقم الصفحة |
| `limit` | int | 50 | عدد المستخدمين لكل صفحة |

**الاستجابة (200):**

```json
{
  "users": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",
      "name": "John Doe",
      "provider": "local",
      "isActive": true,
      "isAdmin": false,
      "accesses": ["user"],
      "createdAt": "2025-01-15 10:30:00"
    },
    {
      "id": "123e4567-e89b-12d3-a456-426614174001",
      "email": "admin@example.com",
      "name": "Admin User",
      "provider": "local",
      "isActive": true,
      "isAdmin": true,
      "accesses": ["user", "superadmin"],
      "createdAt": "2025-01-10 08:00:00"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

### تحديث حالة المستخدم

تفعيل أو تعطيل حساب مستخدم. لا يمكن للمستخدمين المُعطَّلين تسجيل الدخول أو تحديث الرموز.

```
PUT /api/admin/users/:id/status
```

**معاملات المسار:**

| المعامل | الوصف |
|---|---|
| `id` | UUID المستخدم |

**نص الطلب:**

```json
{
  "active": false
}
```

**الاستجابة (200):**

```json
{
  "message": "User status updated successfully"
}
```

---

### تحديث صلاحيات المستخدم

تغيير مصفوفة مستوى الوصول لمستخدم. استخدم هذا للترقية إلى superadmin أو خفض الرتبة أو إضافة وصول moderator.

```
PUT /api/admin/users/:id/accesses
```

**معاملات المسار:**

| المعامل | الوصف |
|---|---|
| `id` | UUID المستخدم |

**نص الطلب:**

```json
{
  "accesses": ["user", "superadmin"]
}
```

| الحقل | النوع | الوصف |
|---|---|---|
| `accesses` | string[] | مصفوفة مستويات الوصول. القيم الصالحة: `user`، `superadmin`، `admin`، `moderator`، `guest` |

**الاستجابة (200):**

```json
{
  "message": "User accesses updated"
}
```

---

### حذف مستخدم

حذف مستخدم وجميع البيانات المرتبطة به نهائياً. يعمل بشكل غير متزامن — يُرجع `202 Accepted` فوراً. لا يمكنك حذف حسابك الخاص.

```
DELETE /api/admin/users/:id
```

**معاملات المسار:**

| المعامل | الوصف |
|---|---|
| `id` | UUID المستخدم |

**الاستجابة (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| الحقل | الوصف |
|---|---|
| `message` | تأكيد أن الحذف قيد التقدم |
| `rooms` | عدد الغرف التي أنشأها هذا المستخدم (محذوفة أيضاً) |

المهمة الخلفية تنفذ حذفاً متسلسلاً عبر:
- جميع الغرف التي أنشأها المستخدم
- مفاتيح المرور المرتبطة (بيانات WebAuthn)
- تفضيلات المستخدم
- سجل المستخدم نفسه

## إدارة الغرف

### عرض جميع الغرف

استرجاع قائمة مقسمة لصفحات بجميع الغرف على النسخة (يتجاوز فلتر المُنشئ).

```
GET /api/admin/rooms?page=1&limit=50
```

**معاملات الاستعلام:**

| المعامل | النوع | الافتراضي | الوصف |
|---|---|---|---|
| `page` | int | 1 | رقم الصفحة |
| `limit` | int | 50 | عدد الغرف لكل صفحة |

**الاستجابة (200):**

```json
{
  "rooms": [
    {
      "id": "room-uuid-1",
      "name": "team-standup",
      "createdBy": "user-uuid",
      "maxParticipants": 25,
      "isActive": true,
      "settings": {
        "allowChat": true,
        "allowAudio": true,
        "allowVideo": true,
        "isPersistent": false
      },
      "createdAt": "2025-03-01T10:00:00Z"
    }
  ],
  "total": 8,
  "page": 1,
  "limit": 50
}
```

### تحديث غرفة

تعديل إعدادات أي غرفة، بما في ذلك علامة `isPersistent` الخاصة بالمشرف فقط.

```
PUT /api/admin/rooms/:roomId
```

**معاملات المسار:**

| المعامل | الوصف |
|---|---|
| `roomId` | UUID الغرفة |

**نص الطلب:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| الحقل | النوع | الوصف |
|---|---|---|
| `maxParticipants` | int | الحد الأقصى للمشاركين المسموح بهم (اختياري) |
| `settings.allowChat` | bool | تفعيل الدردشة داخل الغرفة |
| `settings.allowAudio` | bool | تفعيل مسارات الصوت |
| `settings.allowVideo` | bool | تفعيل مسارات الفيديو |
| `settings.requireApproval` | bool | طلب موافقة المشرف للانضمام |
| `settings.e2ee` | bool | تفعيل التشفير من طرف إلى طرف |
| `settings.isPersistent` | bool | **للمشرف فقط.** منع التنظيف التلقائي للغرف الخاملة |

جميع حقول الإعدادات اختيارية — أدرج فقط الحقول التي تريد تغييرها.

**الاستجابة (200):** تُرجع كائن الغرفة المُحدَّث.

---

### إغلاق غرفة

حذف غرفة وجميع المشاركين فيها نهائياً. يتم وضعها في قائمة انتظار المهام الخلفية وتعود فوراً — سيتم حذف الغرفة بالكامل في غضون ثوانٍ.

```
DELETE /api/admin/rooms/:roomId
```

**معاملات المسار:**

| المعامل | الوصف |
|---|---|
| `roomId` | UUID الغرفة |

**الاستجابة (202):**

```json
{
  "message": "Room deletion queued"
}
```

| الحالة | الوصف |
|--------|-------------|
| 202 | تم وضع الحذف في قائمة الانتظار — سيُكتمل قريباً |
| 404 | الغرفة غير موجودة |
| 500 | فشل في جلب الغرفة أو إدراج المهمة في قائمة الانتظار |

> **ملاحظة:** عند الإغلاق، يتم إرسال رسالة نظام لجميع المشاركين النشطين: "This room has been closed by an administrator".
> منطق الإغلاق يستدعي `cleanupSvc.CascadeDeleteRoom` الذي يتولى إزالة غرفة LiveKit، تنظيف ملفات الدردشة المرفوعة
> وحذف سجل قاعدة البيانات.

---

### تعليق غرفة

إلغاء تنشيط غرفة نشطة، وإنهاء جميع المكالمات النشطة مع الحفاظ على سجل الغرفة. يمكن إعادة تنشيط الغرفة عند انضمام مشارك مرة أخرى. تُنفذ كمهمة خلفية وتعود فوراً.

```
POST /api/admin/rooms/:roomId/suspend
```

**معاملات المسار:**

| المعامل | الوصف |
|---|---|
| `roomId` | UUID الغرفة |

**الاستجابة (202):**

```json
{
  "message": "Room suspend queued"
}
```

| الحالة | الوصف |
|--------|-------------|
| 202 | تم وضع التعليق في قائمة الانتظار — سيُكتمل قريباً |
| 400 | الغرفة غير نشطة بالفعل |
| 404 | الغرفة غير موجودة |
| 500 | فشل في جلب الغرفة أو إدراج المهمة في قائمة الانتظار |

---

### تعليق جماعي للغرف

وضع تعليق لعدة غرف في قائمة الانتظار دفعة واحدة. يتم تخطي الغرف المعلقة بالفعل.

```
POST /api/admin/rooms/bulk/suspend
```

**نص الطلب:**

```json
{
  "ids": ["room-uuid-1", "room-uuid-2"]
}
```

**الاستجابة (202):

```json
{
  "results": {
    "room-uuid-1": {"success": true, "name": "team-standup"},
    "room-uuid-2": {"success": false, "name": "design-review", "error": "room not found"}
  },
  "totalProcessed": 1,
  "totalFailed": 1
}
```

| الحقل | النوع | الوصف |
|-------|------|-------------|
| `results` | object | نتيجة لكل غرفة مفهرسة بالمعرف |
| `results.*.success` | bool | ما إذا تم وضع العملية في قائمة الانتظار |
| `results.*.name` | string | اسم الغرفة (موجود عند النجاح) |
| `results.*.error` | string | رسالة الخطأ (موجودة عند الفشل) |
| `totalProcessed` | int | عدد الغرف التي تمت بنجاح في قائمة الانتظار |
| `totalFailed` | int | عدد الغرف الفاشلة |

### إغلاق جماعي للغرف

وضع حذف دائم لعدة غرف في قائمة الانتظار دفعة واحدة. لا يمكن التراجع عنه.

```
POST /api/admin/rooms/bulk/close
```

نفس تنسيق الطلب/الاستجابة مثل [تعليق جماعي للغرف](#تعليق-جماعي-للغرف).

## رموز الدعوة

تُمكّن رموز الدعوة التسجيل المُقيَّد. عند تفعيل `tokenRegistrationOnly` في الإعدادات، يجب على المستخدمين الجدد تقديم رمز دعوة صالح وغير مستخدم أثناء التسجيل.

### عرض رموز الدعوة

استرجاع جميع رموز الدعوة، الأحدث أولاً. كل رمز يتضمن قيمة `used` المنطقية المحسوبة من `usedAt`.

```
GET /api/admin/invite-tokens
```

### إنشاء رمز دعوة

توليد رمز دعوة جديد. قيمة الرمز هي سلسلة هيكس عشوائية بطول 32 حرفاً.

```
POST /api/admin/invite-tokens
```

**نص الطلب:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| الحقل | النوع | مطلوب | الافتراضي | الوصف |
|---|---|---|---|---|
| `email` | string | لا | _بدون_ | تقييد الرمز لعنوان البريد الإلكتروني هذا |
| `expiresInHours` | int | لا | 72 | مدة صلاحية الرمز بالساعات |

**الاستجابة (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

---

### حذف رمز دعوة

إبطال رمز دعوة بواسطة معرفه. بعد الحذف، لا يمكن استخدام الرمز للتسجيل.

```
DELETE /api/admin/invite-tokens/:id
```

**معاملات المسار:**

| المعامل | الوصف |
|---|---|
| `id` | UUID رمز الدعوة |

**الاستجابة (200):**

```json
{
  "status": "success"
}
```

### معلومات الشهادة

الحصول على بيانات شهادة TLS بما في ذلك حالة الانتهاء.

```
GET /api/admin/cert-info
```

**الاستجابة (200) — TLS مُفعّل، الشهادة صالحة:**

```json
{
  "enabled": true,
  "status": "valid",
  "daysRemaining": 78,
  "notAfter": "2025-07-30T12:00:00Z",
  "subject": "CN=bedrud.example.com",
  "issuer": "CN=R3,O=Let's Encrypt,C=US",
  "notBefore": "2025-04-01T12:00:00Z",
  "sans": ["bedrud.example.com", "www.bedrud.example.com"]
}
```

**الاستجابة (200) — TLS غير مُفعّل:**

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| الحالة | المعنى |
|---|---|
| `valid` | الشهادة صالحة وليست قريبة من الانتهاء |
| `expiring_soon` | تنتهي الشهادة خلال 30 يوماً |
| `expired` | انتهت صلاحية الشهادة |
| `not_configured` | TLS مُعطّل |
| `error` | ملف الشهادة مفقود أو غير صالح |

## استجابات الخطأ

```json
{
  "error": "forbidden"
}
```

| الحالة | المعنى |
|---|---|
| 400 | إدخال غير صالح (مثلاً: JSON غير سليم، حقول مطلوبة مفقودة، حذف الحساب الخاص، تعليق غرفة غير نشطة) |
| 401 | غير مُصادق — JWT مفقود أو منتهٍ |
| 403 | ليس superadmin — المستخدم يفتقر إلى `superadmin` في مصفوفة `accesses` |
| 404 | المورد غير موجود (مستخدم، غرفة، أو رمز) |
| 409 | تعارض — لا يمكن للمستخدم حذف حسابه الخاص |
| 500 | خطأ داخلي في الخادم |
| 501 | غير مُنفّذ (مثلاً: توليد رمز غرفة المشرف) |
| 503 | معلومات الشهادة غير متاحة (خطأ TLS) |
