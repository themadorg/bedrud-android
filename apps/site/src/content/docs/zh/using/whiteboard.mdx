Bedrud 包含协作白板，房间内所有人可在同一场景上绘制、选择与编辑。

## 启用功能

白板可能位于 **设置 → 实验性 → 共享白板** 之后。开启后，在会议中打开白板控制。每次会话前可能需要确认接受。

## 打开画板

1. 加入会议（真实测试建议用两个浏览器）。
2. 从会议控制打开**白板**叠加层。
3. 在用户 A 的画板上绘制——用户 B 应在约**一秒**内看到笔迹。
4. 在 B 上绘制——A 应看到相同更新。

## 一起编辑

| 操作 | 远端预期 |
|------|----------|
| 绘制笔迹 | 快速出现在他人端 |
| 选择 / 移动 / 删除 | 远端场景更新 |
| 远端光标 / 跟随 | 不应冻结 UI |

## 关闭、重开、重新加入

- **关闭**叠加层再打开——共享场景通常仍在（除非有意拆除）。
- 会话中途**离开**房间——无控制台死循环；重新加入应稳定。
- 迟到者通过同一同步路径收到当前共享状态。

## 工作原理（简要）

白板基于协作画布（Excalidraw 风格工具），以 **Yjs** CRDT 状态通过 **LiveKit 数据通道**同步。媒体（音视频）仍走 WebRTC 轨道；画板更新走可靠/尽力而为数据路径，协作不依赖单独的白板服务器。

自托管者：确保生产构建包含白板 vendor 分块（`bun run build` 构建 web 应用），以免打开画板时静态资源 404。
