如何从 Web 应用进入 Bedrud 房间：预加入预览、连接、基本媒体控制与访问门槛。

## 预加入（欢迎外壳）

媒体连接之前，你通常会看到：

- 显示**名称**
- **麦克风**与**摄像头**预览
- 麦克风、摄像头与扬声器的**设备选择器**（可用时）

加入前先设置好，以便首次连接使用正确设备。

## 加入与连接

1. 点击**加入**（或等效按钮）。
2. 应看到短暂加载外壳，然后是**已连接**的会议 UI——而不是无限转圈。
3. 其他参与者加入后以画面或头像出现。

若连接失败，检查浏览器权限、网络，以及服务器的 LiveKit/WebRTC 端口是否可达。运维人员：见 [WebRTC 连接](/zh/docs/architecture/webrtc-connectivity)。

## 基本控制

| 操作 | 预期行为 |
|------|----------|
| 静音 / 取消静音麦克风 | 其他参与者停止/开始听到你 |
| 摄像头开 / 关 | 你的视频画面为他人更新 |
| 离开房间 | 干净断开；他人不应永久保留僵尸参与者 |
| 重新加入 | 离开后再次加入同一房间可用 |

## 多名参与者

用两个浏览器（或正常 + 隐私模式）确认：

- 双方都能看到对方的画面
- 静音与摄像头状态在远端可见

## 受控房间

房间可能需要**密码**、等待批准，或其他取决于房间设置的访问检查。出现时完成**房间访问对话框**，再进入预加入 / 加入。

某些实例可能允许访客加入（无需账户）——分享链接，访客仅用显示名称进入。若房间已满或容量登记失败，加入会失败（不签发媒体令牌）。访客获得**稳定的签名身份 Cookie**（`bedrud_guest_id`），使房间封禁在再次访问时仍然有效。公开房间的大厅**人数**可以公开；参与者身份列表需要身份验证。
