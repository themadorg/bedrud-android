房间内聊天是侧边面板，用于文字、回应、投票与图片，无需离开通话。

## 打开与发送

1. 从会议控制打开**聊天**面板。
2. 发送文字消息——其他参与者会收到。
3. 打开或关闭面板**不得**中断摄像头或麦克风轨道。

即使突发连发，消息分组与时间戳也应正确。

## 表情

- 对消息**回应**——双方更新回应计数。
- 输入框中的**表情选择器**将表情插入草稿。

## 投票

1. 从聊天创建**投票**。
2. 所有人看到投票气泡。
3. 参与者**投票**；结果更新（若 UI 提供结果弹窗）。

## 图片与文件

- 在服务器的大小与尺寸限制内附加图片。
- 会议客户端会在上传前根据公开设置（`chatUploadMaxBytes`、`chatUploadMaxDimension`）**预检**文件大小和图片尺寸。
- 超大文件或超限尺寸会被明确报错拒绝（先客户端；服务端仍为准 — `POST /api/room/:roomId/chat/upload` 返回 `413` / `400`）。
- 接收方可在 **lightbox** 中打开图片。

**默认值：** 每个文件 10 MiB，最大尺寸 8192×8192（未设置时）。

运营者可通过以下方式设置限制：

- [配置](/zh/docs/getting-started/configuration) 中的 `chat.uploads.maxBytes` / `chat.uploads.maxDimension`
- [管理后台](/zh/docs/guides/admin-dashboard) 的 **Chat Uploads** 选项卡（`chatUploadMaxBytes`、`chatUploadMaxDimension`）
- `chat` 下的每用户 / 全局存储配额（`maxUploadBytesPerUser`、disk/S3 后端）


## 保留策略（客户端）

LiveKit 不会像聊天产品那样在服务器上持久化数据通道聊天。Bedrud 应用**客户端**保留提示：

| 设置 | 默认（典型） | 含义 |
|------|--------------|------|
| `chat.maxMessageCount` | 10000 | 内存 / session storage 中保留的消息上限 |
| `chat.messageTTLHours` | 2160（90 天） | 按年龄修剪 |

环境变量覆盖：`CHAT_MAX_MESSAGE_COUNT`、`CHAT_MESSAGE_TTL_HOURS`。

在房间中途**重新加载**后，历史按产品策略（会话恢复或干净启动）处理，不会崩溃。
