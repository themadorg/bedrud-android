Stage mode switches the room layout so a presenter (or stage set) is featured instead of the normal grid or spotlight view.

## Start stage

1. As host (or a role allowed to control stage), **start stage** from meeting controls.
2. Other participants see the **stage layout** and any **join notifier** designed for stage sessions.
3. The participant strip / sidebar may still show the rest of the room (toggle if present).

## Screen share on stage

- Share your screen while stage is active — others see a **stage screen-share overlay**.
- **Stop sharing** clears the overlay.

For non-stage screen share (normal grid), see [Meeting Controls](/en/docs/using/meeting-controls).

## End stage

**End stage** returns the room to the normal grid or spotlight layout.

## Late joiners

Someone who joins **while stage is already active** should see the correct stage state immediately — not an empty grid forever. If a late joiner sees the wrong layout, refresh once; if it still fails, report it as a regression.

## Tips

- Stage is meant for demos, teaching, and one-to-many presentation flows.
- Combine with [YouTube Watch](/en/docs/using/youtube-watch) or [Whiteboard](/en/docs/using/whiteboard) only when those overlays are designed to coexist; toggle carefully so participants are not stuck under multiple full-screen layers.
