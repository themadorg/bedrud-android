Admin endpoints are under `/api/admin/`. All require the **superadmin** access level and a valid JWT in the `Authorization: Bearer <token>` header.

## Overview

| Endpoint | Method | Description |
|---|---|---|
| `/api/admin/users` | GET | List all users (paginated) |
| `/api/admin/users/:id` | GET | Get user detail with rooms |
| `/api/admin/users/:id/status` | PUT | Activate or deactivate a user |
| `/api/admin/users/:id/accesses` | PUT | Update a user's access levels |
| `/api/admin/users/:id` | DELETE | Permanently delete a user |
| `/api/admin/rooms` | GET | List all rooms (paginated) |
| `/api/admin/rooms/:roomId` | PUT | Update any room's settings |
| `/api/admin/rooms/:roomId` | DELETE | Close a room (cascade delete) |
| `/api/admin/rooms/:roomId/suspend` | POST | Suspend an active room |
| `/api/admin/rooms/:roomId/participants` | GET | Get live participant data from LiveKit |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | Kick any participant from a room |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | Mute any participant's audio |
| `/api/admin/settings` | GET | Get system settings (secrets masked) |
| `/api/admin/settings` | PUT | Update system settings |
| `/api/admin/invite-tokens` | GET | List all invite tokens |
| `/api/admin/invite-tokens` | POST | Create a new invite token |
| `/api/admin/invite-tokens/:id` | DELETE | Delete an invite token |
| `/api/admin/online-count` | GET | Total active participants |
| `/api/admin/livekit/stats` | GET | Aggregate LiveKit server stats |
| `/api/admin/cert-info` | GET | TLS certificate metadata |

## Access Levels

Bedrud uses a multi-level access system. Each user has an `accesses` array that can contain multiple levels simultaneously.

| Access Level | Description | Middleware Protection |
|---|---|---|
| `superadmin` | Full system access. Admin panel, user/room management, settings, invite tokens. | `RequireAccess("superadmin")` on `/api/admin/*` |
| `admin` | Defined but not middleware-enforced. Informational only. | None currently |
| `moderator` | Room-level moderation. Block chat, deafen, spotlight, disable video per-room. | Room-scoped `isRoomModerator()` check |
| `user` | Standard authenticated user. Create/join rooms, manage profile. | `Protected()` on user routes |
| `guest` | Temporary user for public room joins. Guest login only. | N/A |

> **Note:** The `/api/admin/*` route group checks for `superadmin`, not `admin`. Only users with `superadmin` in their `accesses` array can use these endpoints.

## User Management

### List All Users

Retrieve a paginated list of all registered users.

```
GET /api/admin/users?page=1&limit=50
```

**Headers:** `Authorization: Bearer <accessToken>`

**Query Parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `page` | int | 1 | Page number |
| `limit` | int | 50 | Users per page |

**Response (200):**

```json
{
  "users": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",
      "name": "John Doe",
      "provider": "local",
      "isActive": true,
      "isAdmin": false,
      "accesses": ["user"],
      "createdAt": "2025-01-15 10:30:00"
    },
    {
      "id": "123e4567-e89b-12d3-a456-426614174001",
      "email": "admin@example.com",
      "name": "Admin User",
      "provider": "local",
      "isActive": true,
      "isAdmin": true,
      "accesses": ["user", "superadmin"],
      "createdAt": "2025-01-10 08:00:00"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

### Update User Status

Activate or deactivate a user account. Deactivated users cannot log in or refresh tokens.

```
PUT /api/admin/users/:id/status
```

**Path Parameters:**

| Parameter | Description |
|---|---|
| `id` | User UUID |

```json

```

```json

  "message": "User status updated successfully"
}
```

### Update User Accesses

Change a user's access level array. Use this to promote to superadmin, demote, or add moderator access.

```
PUT /api/admin/users/:id/accesses
```

**Path Parameters:**

| Parameter | Description |
|---|---|
| `id` | User UUID |

**Request Body:**

```json
{
  "accesses": ["user", "superadmin"]
}
```

| Field | Type | Description |
|---|---|---|
| `accesses` | string[] | Array of access levels. Valid values: `user`, `superadmin`, `admin`, `moderator`, `guest` |

**Response (200):**

```json
{
  "message": "User accesses updated"
}
```

---

### Delete User

Permanently delete a user and all associated data. Runs asynchronously — returns `202 Accepted` immediately. Cannot delete your own account.

```
DELETE /api/admin/users/:id
```

**Path Parameters:**

| Parameter | Description |
|---|---|
| `id` | User UUID |

**Response (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| Field | Description |
|---|---|
| `message` | Confirmation that deletion is in progress |
| `rooms` | Number of rooms created by this user (also deleted) |

The background job cascades deletion across:
- All rooms created by the user
- Associated passkeys (WebAuthn credentials)
- User preferences
- The user record itself

### Update Room

Modify any room's settings, including the admin-only `isPersistent` flag.

```
PUT /api/admin/rooms/:roomId
```

**Path Parameters:**

| Parameter | Description |
|---|---|
| `roomId` | Room UUID |

**Request Body:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| Field | Type | Description |
|---|---|---|
| `maxParticipants` | int | Maximum participants allowed (optional) |
| `settings.allowChat` | bool | Enable in-room chat |
| `settings.allowAudio` | bool | Enable audio tracks |
| `settings.allowVideo` | bool | Enable video tracks |
| `settings.requireApproval` | bool | Require admin approval to join |
| `settings.e2ee` | bool | Enable end-to-end encryption |
| `settings.isPersistent` | bool | **Admin-only.** Prevent idle cleanup |

All settings fields are optional — only include the ones you want to change.

**Response (200):** Returns the updated room object.

### Suspend Room

Deactivate an active room and all its participants without deleting the room record. The room can be re-activated when a participant joins again.

```
POST /api/admin/rooms/:roomId/suspend
```

**Path Parameters:**

| Parameter | Description |
|---|---|
| `roomId` | Room UUID |

**Response (200):**

```json
{
  "status": "success"
}
```

**Error (400):** Returns `{"error": "Room is not active"}` if the room is already inactive.

### Update Settings

Update system settings. Secret fields can be sent as `"••••••••"` or empty string to preserve the existing value — the server will only overwrite secrets when a real value is sent.

```
PUT /api/admin/settings
```

**Request Body (partial example):**

```json
{
  "registrationEnabled": false,
  "tokenRegistrationOnly": true,
  "passkeysEnabled": true,
  "googleClientId": "your-client-id.apps.googleusercontent.com",
  "googleClientSecret": "GOCSPX-real-secret-value",
  "googleRedirectUrl": "https://your-server.com/api/auth/google/callback",
  "livekitHost": "127.0.0.1:7880",
  "livekitApiKey": "APILiveKitKey123",
  "livekitApiSecret": "real-livekit-secret",
  "logLevel": "debug"
}
```

**Response (200):** Returns the full settings object with secrets masked.

On save, the server automatically reloads OAuth providers and CORS configuration.

> **Secret fields:** `googleClientSecret`, `githubClientSecret`, `twitterClientSecret`, `jwtSecret`, `sessionSecret`, `livekitApiSecret`, `chatUploadS3SecretKey`

## Invite Tokens

Invite tokens enable gated registration. When `tokenRegistrationOnly` is enabled in settings, new users must provide a valid, unused invite token during registration.

### List Invite Tokens

Retrieve all invite tokens, newest first. Each token includes a `used` boolean computed from `usedAt`.

```
GET /api/admin/invite-tokens
```

**Response (200):**

```json
{
  "tokens": [
    {
      "id": "token-uuid-1",
      "token": "a1b2c3d4e5f6...",
      "email": "invited@example.com",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-06-15T10:30:00Z",
      "usedAt": null,
      "usedBy": "",
      "createdAt": "2025-03-15T10:30:00Z",
      "used": false
    },
    {
      "id": "token-uuid-2",
      "token": "f6e5d4c3b2a1...",
      "email": "",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-04-15T10:30:00Z",
      "usedAt": "2025-03-20T14:00:00Z",
      "usedBy": "new-user-uuid",
      "createdAt": "2025-03-12T10:30:00Z",
      "used": true
    }
  ]
}
```

### Create Invite Token

Generate a new invite token. The token value is a 32-character random hex string.

```
POST /api/admin/invite-tokens
```

**Request Body:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `email` | string | No | _none_ | Restrict token to this email address |
| `expiresInHours` | int | No | 72 | Token validity period in hours |

**Response (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

---

**Path Parameters:**

| Parameter | Description |
|---|---|

### Certificate Info

Get TLS certificate metadata including expiry status.

```
GET /api/admin/cert-info
```

**Response (200) — TLS enabled, certificate valid:**

```json
{
  "enabled": true,
  "status": "valid",
  "daysRemaining": 78,
  "notAfter": "2025-07-30T12:00:00Z",
  "subject": "CN=bedrud.example.com",
  "issuer": "CN=R3,O=Let's Encrypt,C=US",
  "notBefore": "2025-04-01T12:00:00Z",
  "sans": ["bedrud.example.com", "www.bedrud.example.com"]
}
```

**Response (200) — TLS not enabled:**

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| Status | Meaning |
|---|---|
| `valid` | Certificate is valid and not near expiry |
| `expiring_soon` | Certificate expires within 30 days |
| `expired` | Certificate has expired |
| `not_configured` | TLS is disabled |
| `error` | Certificate file missing or invalid |

## Error Responses

```json
{
  "error": "forbidden"
}
```

| Status | Meaning |
|---|---|
| 400 | Invalid input (e.g., bad JSON, missing required fields, deleting own account, suspending inactive room) |
| 401 | Not authenticated — missing or expired JWT |
| 403 | Not a superadmin — user lacks `superadmin` in their `accesses` array |
| 404 | Resource not found (user, room, or token) |
| 409 | Conflict — user cannot delete their own account |
| 500 | Internal server error |
| 501 | Not implemented (e.g., admin room token generation) |
| 503 | Certificate information unavailable (TLS error) |
