Эндпоинты администратора находятся по пути `/api/admin/`. Все требуют уровня доступа **superadmin** и действительный JWT-токен в заголовке `Authorization: Bearer <token>`.

## Обзор

| Эндпоинт | Метод | Описание |
|---|---|---|
| `/api/admin/users` | GET | Список всех пользователей (с пагинацией) |
| `/api/admin/users/:id` | GET | Получить данные пользователя с комнатами |
| `/api/admin/users/:id/status` | PUT | Активировать или деактивировать пользователя |
| `/api/admin/users/:id/accesses` | PUT | Обновить уровни доступа пользователя |
| `/api/admin/users/:id` | DELETE | Безвозвратно удалить пользователя |
| `/api/admin/rooms` | GET | Список всех комнат (с пагинацией) |
| `/api/admin/rooms/:roomId` | PUT | Изменить настройки любой комнаты |
| `/api/admin/rooms/:roomId` | DELETE | Закрыть комнату (каскадное удаление) |
| `/api/admin/rooms/:roomId/suspend` | POST | Приостановить активную комнату |
| `/api/admin/rooms/:roomId/participants` | GET | Получить данные об участниках из LiveKit |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | Выгнать любого участника из комнаты |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | Заглушить аудио любого участника |
| `/api/admin/settings` | GET | Получить системные настройки (секреты скрыты) |
| `/api/admin/settings` | PUT | Обновить системные настройки |
| `/api/admin/invite-tokens` | GET | Список всех токенов приглашения |
| `/api/admin/invite-tokens` | POST | Создать новый токен приглашения |
| `/api/admin/invite-tokens/:id` | DELETE | Удалить токен приглашения |
| `/api/admin/online-count` | GET | Общее количество активных участников |
| `/api/admin/livekit/stats` | GET | Агрегированная статистика сервера LiveKit |
| `/api/admin/cert-info` | GET | Метаданные TLS-сертификата |

## Уровни доступа

Bedrud использует многоуровневую систему доступа. Каждый пользователь имеет массив `accesses`, который может одновременно содержать несколько уровней.

| Уровень доступа | Описание | Защита middleware |
|---|---|---|
| `superadmin` | Полный доступ к системе. Панель администратора, управление пользователями и комнатами, настройки, токены приглашения. | `RequireAccess("superadmin")` на `/api/admin/*` |
| `admin` | Определён, но не применяется middleware. Только информационный. | В настоящее время отсутствует |
| `moderator` | Модерация на уровне комнаты. Блокировка чата, заглушение, spotlight, отключение видео для конкретной комнаты. | Проверка `isRoomModerator()` в контексте комнаты |
| `user` | Стандартный авторизованный пользователь. Создание и вход в комнаты, управление профилем. | `Protected()` на пользовательских маршрутах |
| `guest` | Временный пользователь для входа в публичные комнаты. Только гостевой вход. | Н/Д |

> **Примечание:** Группа маршрутов `/api/admin/*` проверяет наличие `superadmin`, а не `admin`. Только пользователи с `superadmin` в массиве `accesses` могут использовать эти эндпоинты.

## Управление пользователями

### Список всех пользователей

Получить список всех зарегистрированных пользователей с пагинацией.

```
GET /api/admin/users?page=1&limit=50
```

**Заголовки:** `Authorization: Bearer <accessToken>`

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `page` | int | 1 | Номер страницы |
| `limit` | int | 50 | Пользователей на странице |

**Ответ (200):**

```json
{
  "users": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",
      "name": "John Doe",
      "provider": "local",
      "isActive": true,
      "isAdmin": false,
      "accesses": ["user"],
      "createdAt": "2025-01-15 10:30:00"
    },
    {
      "id": "123e4567-e89b-12d3-a456-426614174001",
      "email": "admin@example.com",
      "name": "Admin User",
      "provider": "local",
      "isActive": true,
      "isAdmin": true,
      "accesses": ["user", "superadmin"],
      "createdAt": "2025-01-10 08:00:00"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

Получить профиль конкретного пользователя и созданные им комнаты.

```
GET /api/admin/users/:id
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `id` | UUID пользователя |

**Ответ (200):**

```json
{
  "user": {
    "id": "123e4567-e89b-12d3-a456-426614174000",
    "email": "user@example.com",
    "name": "John Doe",
    "provider": "local",
    "isActive": true,
    "isAdmin": false,
    "accesses": ["user"],
    "createdAt": "2025-01-15 10:30:00"
  },
  "rooms": [
    {
      "id": "room-uuid-1",
      "name": "team-standup",
      "createdAt": "2025-02-01T09:00:00Z"
    }
  ]
}
```

### Обновить статус пользователя

Активировать или деактивировать учётную запись пользователя. Деактивированные пользователи не могут войти или обновить токен.

```
PUT /api/admin/users/:id/status
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `id` | UUID пользователя |

**Тело запроса:**

```json
{
  "active": false
}
```

**Ответ (200):**

```json
{
  "message": "User status updated successfully"
}
```

---

### Обновить уровни доступа пользователя

Изменить массив уровней доступа пользователя. Используйте для повышения до суперадминистратора, понижения или добавления доступа модератора.

```
PUT /api/admin/users/:id/accesses
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `id` | UUID пользователя |

**Тело запроса:**

```json
{
  "accesses": ["user", "superadmin"]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `accesses` | string[] | Массив уровней доступа. Допустимые значения: `user`, `superadmin`, `admin`, `moderator`, `guest` |

**Ответ (200):**

```json
{
  "message": "User accesses updated"
}
```

---

### Удалить пользователя

Безвозвратно удалить пользователя и все связанные данные. Выполняется асинхронно — сразу возвращает `202 Accepted`. Нельзя удалить свою учётную запись.

```
DELETE /api/admin/users/:id
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `id` | UUID пользователя |

**Ответ (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| Поле | Описание |
|---|---|
| `message` | Подтверждение, что удаление в процессе |
| `rooms` | Количество комнат, созданных пользователем (также будут удалены) |

Фоновое задание выполняет каскадное удаление:
- Все комнаты, созданные пользователем
- Связанные passkeys (учётные данные WebAuthn)
- Настройки пользователя
- Сама запись пользователя

## Управление комнатами

### Список всех комнат

Получить список всех комнат экземпляра с пагинацией (в обход фильтра по создателю).

```
GET /api/admin/rooms?page=1&limit=50
```

**Параметры запроса:**

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `page` | int | 1 | Номер страницы |
| `limit` | int | 50 | Комнат на странице |

**Ответ (200):**

```json
{
  "rooms": [
    {
      "id": "room-uuid-1",
      "name": "team-standup",
      "createdBy": "user-uuid",
      "maxParticipants": 25,
      "isActive": true,
      "settings": {
        "allowChat": true,
        "allowAudio": true,
        "allowVideo": true,
        "isPersistent": false
      },
      "createdAt": "2025-03-01T10:00:00Z"
    }
  ],
  "total": 8,
  "page": 1,
  "limit": 50
}
```

---

### Обновить комнату

Изменить настройки любой комнаты, включая флаг `isPersistent`, доступный только администратору.

```
PUT /api/admin/rooms/:roomId
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `roomId` | UUID комнаты |

**Тело запроса:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| Поле | Тип | Описание |
|---|---|---|
| `maxParticipants` | int | Максимальное количество участников (опционально) |
| `settings.allowChat` | bool | Включить чат в комнате |
| `settings.allowAudio` | bool | Включить аудиодорожки |
| `settings.allowVideo` | bool | Включить видеодорожки |
| `settings.requireApproval` | bool | Требовать одобрение для входа |
| `settings.e2ee` | bool | Включить сквозное шифрование |
| `settings.isPersistent` | bool | **Только для администратора.** Предотвратить автоматическую очистку неактивных комнат |

Все поля настроек опциональны — укажите только те, которые нужно изменить.

**Ответ (200):** Возвращает обновлённый объект комнаты.

---

### Закрыть комнату

Безвозвратно закрыть комнату с каскадным удалением. Отправляет системное сообщение всем участникам: «This room has been closed by an administrator».

```
DELETE /api/admin/rooms/:roomId
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `roomId` | UUID комнаты |

**Ответ (200):**

```json
{
  "status": "success"
}
```

---

### Приостановить комнату

Деактивировать активную комнату и всех её участников без удаления записи комнаты. Комнату можно повторно активировать при подключении нового участника.

```
POST /api/admin/rooms/:roomId/suspend
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `roomId` | UUID комнаты |

**Ответ (200):**

```json
{
  "status": "success"
}
```

**Ошибка (400):** Возвращает `{"error": "Room is not active"}`, если комната уже неактивна.

---

**Ответ (200):**

```json
      "sid": "PA_abc123",
      "identity": "user-uuid",
      "name": "John Doe",
      "state": "ACTIVE",
      "joinedAt": 1710338400,
      "isPublisher": true,
      "tracks": [
        {
          "sid": "TR_audio1",
          "type": "AUDIO",
          "source": "MICROPHONE",

          "sid": "TR_video1",
```

---

### Выгнать участника

Удалить любого участника из комнаты (без проверки прав создателя).

```
POST /api/admin/rooms/:roomId/participants/:identity/kick
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `roomId` | UUID комнаты |
| `identity` | Идентификатор участника (UUID пользователя) |

**Ответ (200):**

```json
{
  "status": "success"
}
```

---

### Заглушить участника

Заглушить все аудиодорожки любого участника в комнате.

```
POST /api/admin/rooms/:roomId/participants/:identity/mute
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `roomId` | UUID комнаты |
| `identity` | Идентификатор участника (UUID пользователя) |

**Тело запроса:**

```json

  "muted": true

```

**Ответ (200):**

```json
{
  "status": "success"
}
```

```
PUT /api/admin/settings
```

**Тело запроса (частичный пример):**

```json
{
  "registrationEnabled": false,
  "tokenRegistrationOnly": true,
  "passkeysEnabled": true,
  "googleClientId": "your-client-id.apps.googleusercontent.com",
```

## Токены приглашения

Токены приглашения включают ограниченную регистрацию. Когда в настройках включён `tokenRegistrationOnly`, новые пользователи должны предоставить действительный неиспользованный токен приглашения при регистрации.

### Список токенов приглашения

Получить все токены приглашения, начиная с новейших. Каждый токен включает вычисляемое поле `used` на основе `usedAt`.

```
GET /api/admin/invite-tokens
```

**Ответ (200):**

```json
{
  "tokens": [
    {
      "id": "token-uuid-1",
      "token": "a1b2c3d4e5f6...",
      "email": "invited@example.com",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-06-15T10:30:00Z",
      "usedAt": null,
      "usedBy": "",
      "createdAt": "2025-03-15T10:30:00Z",
      "used": false
    },
    {
      "id": "token-uuid-2",
      "token": "f6e5d4c3b2a1...",
      "email": "",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-04-15T10:30:00Z",
      "usedAt": "2025-03-20T14:00:00Z",
      "usedBy": "new-user-uuid",
      "createdAt": "2025-03-12T10:30:00Z",
      "used": true
    }
  ]
}
```

---

### Создать токен приглашения

Сгенерировать новый токен приглашения. Значение токена — случайная шестнадцатеричная строка из 32 символов.

```
POST /api/admin/invite-tokens
```

**Тело запроса:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| Поле | Тип | Обязательное | По умолчанию | Описание |
|---|---|---|---|---|
| `email` | string | Нет | _нет_ | Ограничить токен этим адресом email |
| `expiresInHours` | int | Нет | 72 | Срок действия токена в часах |

**Ответ (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

---

### Удалить токен приглашения

Отозвать токен приглашения по его ID. После удаления токен больше не может использоваться для регистрации.

```
DELETE /api/admin/invite-tokens/:id
```

**Параметры пути:**

| Параметр | Описание |
|---|---|
| `id` | UUID токена приглашения |

**Ответ (200):**

```json
{
  "status": "success"
}
```

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| Статус | Значение |
|---|---|
| `valid` | Сертификат действителен и не близок к истечению |
| `expiring_soon` | Сертификат истекает в течение 30 дней |
| `expired` | Срок действия сертификата истёк |
| `not_configured` | TLS отключён |
| `error` | Файл сертификата отсутствует или недействителен |

---

## Ответы об ошибках

```json
{
  "error": "forbidden"
}
```

| Статус | Значение |
|---|---|
| 400 | Некорректный ввод (например, неверный JSON, отсутствуют обязательные поля, попытка удалить свою учётную запись, приостановка неактивной комнаты) |
| 401 | Не авторизован — отсутствует или истёк JWT-токен |
| 403 | Не суперадминистратор — у пользователя отсутствует `superadmin` в массиве `accesses` |
| 404 | Ресурс не найден (пользователь, комната или токен) |
| 409 | Конфликт — пользователь не может удалить свою учётную запись |
| 500 | Внутренняя ошибка сервера |
| 501 | Не реализовано (например, генерация токена комнаты администратором) |
| 503 | Информация о сертификате недоступна (ошибка TLS) |
