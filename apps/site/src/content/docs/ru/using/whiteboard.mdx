В Bedrud есть совместная доска: все в комнате могут рисовать, выделять и редактировать одну и ту же сцену.

## Включение функции

Доска может быть за **Настройки → Экспериментальное → Shared whiteboard**. Включите, затем откройте контрол доски во встрече. Перед каждой сессией может потребоваться подтверждение.

## Открытие доски

1. Войдите на встречу (для реального теста рекомендуются два браузера).
2. Откройте оверлей **whiteboard** из контролов встречи.
3. Рисуйте на доске пользователя A — пользователь B должен увидеть штрихи примерно за **одну секунду**.
4. Рисуйте на B — A должен увидеть те же обновления.

## Совместное редактирование

| Действие | Ожидание удалённо |
|----------|-------------------|
| Штрихи | Быстро появляются у других |
| Выделение / перенос / удаление | Удалённая сцена обновляется |
| Удалённые курсоры / follow | Не должны замораживать UI |

## Закрытие, повторное открытие, rejoin

- **Закройте** оверлей и откройте снова — общая сцена обычно остаётся (если её намеренно не сбросили).
- **Выйдите** из комнаты mid-session — без console death loop; rejoin должен быть стабильным.
- Поздние входы получают текущее общее состояние по тому же пути синхронизации.

## Как это работает (кратко)

Доска построена на collaborative canvas (инструменты в стиле Excalidraw) с CRDT-состоянием **Yjs**, синхронизируемым по **LiveKit data channels**. Медиа (аудио/видео) остаётся на WebRTC tracks; обновления доски идут по reliable/lossy data paths, поэтому коллаборация не зависит от отдельного whiteboard-сервера.

Self-hosters: убедитесь, что production build включает vendor chunk доски (`bun run build` для web app), чтобы открытие доски не давало 404 на статические ассеты.
