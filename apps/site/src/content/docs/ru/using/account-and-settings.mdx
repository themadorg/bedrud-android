Создайте аккаунт, сохраните сессию и настройте профиль и медиа до входа в звонок.

## Регистрация и вход

1. Откройте ваш экземпляр Bedrud в браузере.
2. **Зарегистрируйтесь** с email и паролем (и токеном приглашения, если его требует хост).
3. Если включена проверка email, завершите шаг подтверждения, когда UI запросит.
4. **Войдите** с верными данными. Неверный пароль показывает понятную ошибку и не создаёт сессию.
5. **Выйдите** из меню аккаунта. Защищённые маршруты (дашборд, настройки, админка) перенаправляют на авторизацию.

Сессии переживают полную **перезагрузку** страницы, пока access-токен действителен.

### Другие способы входа

В зависимости от конфигурации экземпляра вы также можете увидеть:

- **OAuth** (Google, GitHub, Twitter или свой OIDC)
- **Passkeys** (WebAuthn) после регистрации в настройках безопасности
- **Гостевой вход** для комнат, где это разрешено — без аккаунта (см. [Вход на встречу](/ru/docs/using/joining-a-meeting))

Забыли пароль? Используйте **Forgot password** на странице входа и перейдите по ссылке сброса (на сервере должен быть настроен SMTP).

Подробности для операторов (JWT, passkeys, rate limits): [Аутентификация](/ru/docs/backend/authentication).

## Профиль

Откройте **Настройки → Профиль** с дашборда.

- Измените **имя** / поля отображения и сохраните — значения корректно подгружаются после перезагрузки.
- Изменения видны на дашборде, плитках встречи и в списках участников.

## Аватар

Всё ещё в **Настройки → Профиль**:

| Правило | Детали |
|---------|--------|
| Форматы | JPEG, PNG, WebP |
| Лимит размера | Меньше **2 МБ** |
| Перегруз / неверный MIME | Отклоняется с ошибкой (без краша) |
| Повторная загрузка | Заменяет предыдущий файл |

После загрузки превью обновляется сразу. Подтвердите новый аватар после перезагрузки на дашборде и во встрече.

## Панели настроек

Откройте каждую панель с маршрута настроек дашборда:

| Панель | Назначение |
|--------|------------|
| **Профиль** | Имя и отображаемая личность |
| **Аудио** | Списки микрофона / динамика; включение **push-to-talk** + захват hold-клавиши; сохраняется после перезагрузки |
| **Видео** | Выбор камеры; сохраняется после перезагрузки |
| **Безопасность** | Смена пароля и регистрация passkey |
| **Внешний вид** | Тема и связанные переключатели UI (если показаны) |
| **Экспериментальное** | Опциональные функции: **общая доска** и **YouTube watch party** |

Предпочтения хранятся для аккаунта (и могут синхронизироваться в оболочку встречи при входе). Поведение PTT во время звонка: [Аудио и Push-to-Talk](/ru/docs/using/audio-and-push-to-talk).
