نقاط پایانی مدیریت تحت `/api/admin/` قرار دارند. تمامی این نقاط نیازمند سطح دسترسی **superadmin** و یک JWT معتبر در هدر `Authorization: Bearer <token>` می‌باشند.

## نمای کلی

| نقطه پایانی | روش | توضیحات |
|---|---|---|
| `/api/admin/users` | GET | لیست تمام کاربران (صفحه‌بندی شده) |
| `/api/admin/users/:id` | GET | جزئیات کاربر به همراه اتاق‌ها |
| `/api/admin/users/:id/status` | PUT | فعال یا غیرفعال کردن یک کاربر |
| `/api/admin/users/:id/accesses` | PUT | به‌روزرسانی سطوح دسترسی یک کاربر |
| `/api/admin/users/:id` | DELETE | حذف دائمی یک کاربر |
| `/api/admin/rooms` | GET | لیست تمام اتاق‌ها (صفحه‌بندی شده) |
| `/api/admin/rooms/:roomId` | PUT | به‌روزرسانی تنظیمات هر اتاق |
| `/api/admin/rooms/:roomId` | DELETE | حذف یک اتاق (کار پس‌زمینه) |
| `/api/admin/rooms/:roomId/suspend` | POST | تعلیق یک اتاق (کار پس‌زمینه) |
| `/api/admin/rooms/bulk/suspend` | POST | تعلیق گروهی اتاق‌ها |
| `/api/admin/rooms/bulk/close` | POST | بستن گروهی اتاق‌ها |
| `/api/admin/users/bulk/delete` | POST | حذف گروهی کاربران |
| `/api/admin/rooms/:roomId/participants` | GET | دریافت داده‌های زنده شرکت‌کنندگان از LiveKit |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | اخراج هر شرکت‌کننده از اتاق |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | قطع صدای هر شرکت‌کننده |
| `/api/admin/settings` | GET | دریافت تنظیمات سیستم (مقادیر محرمانه پوشانده شده) |
| `/api/admin/settings` | PUT | به‌روزرسانی تنظیمات سیستم |
| `/api/admin/invite-tokens` | GET | لیست تمام توکن‌های دعوت |
| `/api/admin/invite-tokens` | POST | ایجاد یک توکن دعوت جدید |
| `/api/admin/invite-tokens/:id` | DELETE | حذف یک توکن دعوت |
| `/api/admin/online-count` | GET | تعداد کل شرکت‌کنندگان فعال |
| `/api/admin/livekit/stats` | GET | آمار تجمیعی سرور LiveKit |
| `/api/admin/cert-info` | GET | فراداده گواهی TLS |

## سطوح دسترسی

Bedrud از یک سیستم دسترسی چندسطحی استفاده می‌کند. هر کاربر دارای یک آرایه `accesses` است که می‌تواند همزمان شامل چندین سطح دسترسی باشد.

| سطح دسترسی | توضیحات | محافظت میان‌افزار |
|---|---|---|
| `superadmin` | دسترسی کامل سیستم. پنل مدیریت، مدیریت کاربران/اتاق‌ها، تنظیمات، توکن‌های دعوت. | `RequireAccess("superadmin")` روی `/api/admin/*` |
| `admin` | تعریف شده اما توسط میان‌افزار اعمال نمی‌شود. فقط اطلاع‌رسانی. | در حال حاضر هیچ‌کدام |
| `moderator` | مدیریت در سطح اتاق. مسدود کردن چت، قطع صدا، اسپات‌لایت، غیرفعال کردن ویدیو در هر اتاق. | بررسی `isRoomModerator()` در محدوده اتاق |
| `user` | کاربر احراز هویت شده استاندارد. ایجاد/پیوستن به اتاق‌ها، مدیریت پروفایل. | `Protected()` روی مسیرهای کاربر |
| `guest` | کاربر موقت برای پیوستن به اتاق‌های عمومی. فقط ورود مهمان. | ندارد |

> **توجه:** گروه مسیر `/api/admin/*` سطح `superadmin` را بررسی می‌کند، نه `admin` را. فقط کاربرانی که `superadmin` در آرایه `accesses` خود دارند می‌توانند از این نقاط پایانی استفاده کنند.

## مدیریت کاربران

### لیست تمام کاربران

دریافت لیست صفحه‌بندی شده تمام کاربران ثبت‌نام شده.

```
GET /api/admin/users?page=1&limit=50
```

**هدرها:** `Authorization: Bearer <accessToken>`

**پارامترهای پرسمان:**

| پارامتر | نوع | پیش‌فرض | توضیحات |
|---|---|---|---|
| `page` | int | 1 | شماره صفحه |
| `limit` | int | 50 | تعداد کاربران در هر صفحه |

**پاسخ (200):**

```json
{
  "users": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",
      "name": "John Doe",
      "provider": "local",
      "isActive": true,
      "isAdmin": false,
      "accesses": ["user"],
      "createdAt": "2025-01-15 10:30:00"
    },
    {
      "id": "123e4567-e89b-12d3-a456-426614174001",
      "email": "admin@example.com",
      "name": "Admin User",
      "provider": "local",
      "isActive": true,
      "isAdmin": true,
      "accesses": ["user", "superadmin"],
      "createdAt": "2025-01-10 08:00:00"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

### به‌روزرسانی وضعیت کاربر

فعال یا غیرفعال کردن یک حساب کاربری. کاربران غیرفعال نمی‌توانند وارد شوند یا توکن خود را تازه‌سازی کنند.

```
PUT /api/admin/users/:id/status
```

**پارامترهای مسیر:**

| پارامتر | توضیحات |
|---|---|
| `id` | UUID کاربر |

**بدنه درخواست:**

```json
{
  "active": false
}
```

**پاسخ (200):**

```json
{
  "message": "User status updated successfully"
}
```

---

### به‌روزرسانی سطوح دسترسی کاربر

تغییر آرایه سطوح دسترسی یک کاربر. برای ارتقا به superadmin، تنزل رتبه، یا افزودن دسترسی moderator استفاده می‌شود.

```
PUT /api/admin/users/:id/accesses
```

**پارامترهای مسیر:**

| پارامتر | توضیحات |
|---|---|
| `id` | UUID کاربر |

**بدنه درخواست:**

```json
{
  "accesses": ["user", "superadmin"]
}
```

| فیلد | نوع | توضیحات |
|---|---|---|
| `accesses` | string[] | آرایه سطوح دسترسی. مقادیر معتبر: `user`، `superadmin`، `admin`، `moderator`، `guest` |

**پاسخ (200):**

```json
{
  "message": "User accesses updated"
}
```

### حذف کاربر

حذف دائمی یک کاربر و تمام داده‌های مرتبط. به صورت ناهمگام اجرا می‌شود — بلافاصله `202 Accepted` برمی‌گرداند. نمی‌توانید حساب خودتان را حذف کنید.

```
DELETE /api/admin/users/:id
```

**پارامترهای مسیر:**

| پارامتر | توضیحات |
|---|---|
| `id` | UUID کاربر |

**پاسخ (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| فیلد | توضیحات |
|---|---|
| `message` | تأیید در حال انجام بودن عملیات حذف |
| `rooms` | تعداد اتاق‌های ایجاد شده توسط این کاربر (که آن‌ها نیز حذف می‌شوند) |

کار پس‌زمینه، حذف آبشاری را در موارد زیر انجام می‌دهد:
- تمام اتاق‌های ایجاد شده توسط کاربر
- کلیدهای عبور مرتبط (اعتبارنامه‌های WebAuthn)
- تنظیمات کاربر
- خود record کاربر

## مدیریت اتاق‌ها

### لیست تمام اتاق‌ها

```
GET /api/admin/rooms?page=1&limit=50
```

**پارامترهای پرسمان:**

| پارامتر | نوع | پیش‌فرض | توضیحات |

| `limit` | int | 50 | تعداد اتاق‌ها در هر صفحه |

**پاسخ (200):**

```json
{
  "rooms": [
    {
      "id": "room-uuid-1",
      "name": "team-standup",
      "createdBy": "user-uuid",
      "maxParticipants": 25,
      "isActive": true,
      "settings": {
        "allowChat": true,
        "allowAudio": true,
```

### به‌روزرسانی اتاق

تغییر تنظیمات هر اتاق، از جمله پرچم `isPersistent` که فقط برای `superadmin` قابل تنظیم است.

```
PUT /api/admin/rooms/:roomId
```

**پارامترهای مسیر:**

| پارامتر | توضیحات |
|---|---|
| `roomId` | UUID اتاق |

**بدنه درخواست:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| فیلد | نوع | توضیحات |
|---|---|---|
| `maxParticipants` | int | حداکثر تعداد شرکت‌کنندگان مجاز (اختیاری) |
| `settings.allowChat` | bool | فعال کردن چت درون اتاق |
| `settings.allowAudio` | bool | فعال کردن ترک‌های صوتی |
| `settings.allowVideo` | bool | فعال کردن ترک‌های تصویری |
| `settings.requireApproval` | bool | نیاز به تأیید `superadmin` برای پیوستن |
| `settings.e2ee` | bool | فعال کردن رمزگذاری سرتاسری |
| `settings.isPersistent` | bool | **فقط `superadmin`.** جلوگیری از پاکسازی خودکار اتاق‌های بیکار |

تمامی فیلدهای تنظیمات اختیاری هستند — فقط مواردی را که می‌خواهید تغییر دهید ارسال کنید.

**پاسخ (200):** شیء اتاق به‌روزرسانی شده را برمی‌گرداند.

---

### بستن اتاق

حذف دائمی یک اتاق و تمام شرکت‌کنندگان آن. در یک کار پس‌زمینه قرار می‌گیرد و بلافاصله برمی‌گردد — اتاق ظرف چند ثانیه کاملاً حذف می‌شود.

```
DELETE /api/admin/rooms/:roomId
```

**پارامترهای مسیر:**

| پارامتر | توضیحات |
|---|---|
| `roomId` | UUID اتاق |

**پاسخ (202):**

```json
{
  "message": "Room deletion queued"
}
```

| وضعیت | توضیحات |
|--------|-------------|
| 202 | حذف در صف قرار گرفت — به زودی تکمیل می‌شود |
| 404 | اتاق پیدا نشد |
| 500 | دریافت اتاق یا قرار دادن در صف ناموفق بود |

> **توجه:** بستن یک پیام سیستمی به تمام شرکت‌کنندگان فعال ارسال می‌کند: "This room has been closed by an administrator".
> منطق بستن `cleanupSvc.CascadeDeleteRoom` را فراخوانی می‌کند که حذف اتاق LiveKit، پاکسازی آپلودهای چت
> و حذف رکورد پایگاه داده را انجام می‌دهد.

---

### تعلیق اتاق

غیرفعال کردن یک اتاق فعال، پایان دادن به تمام تماس‌های فعال اما حفظ رکورد اتاق. اتاق زمانی که یک شرکت‌کننده دوباره بپیوندد می‌تواند مجدداً فعال شود. به عنوان یک کار پس‌زمینه اجرا می‌شود و بلافاصله برمی‌گردد.

```
POST /api/admin/rooms/:roomId/suspend
```

**پارامترهای مسیر:**

| پارامتر | توضیحات |
|---|---|
| `roomId` | UUID اتاق |

**پاسخ (202):**

```json
{
  "message": "Room suspend queued"
}
```

| وضعیت | توضیحات |
|--------|-------------|
| 202 | تعلیق در صف قرار گرفت — به زودی تکمیل می‌شود |
| 400 | اتاق قبلاً غیرفعال است |
| 404 | اتاق پیدا نشد |
| 500 | دریافت اتاق یا قرار دادن در صف ناموفق بود |

---

### تعلیق گروهی اتاق‌ها

قرار دادن تعلیق برای چندین اتاق به طور همزمان در صف. اتاق‌های قبلاً تعلیق شده نادیده گرفته می‌شوند.

```
POST /api/admin/rooms/bulk/suspend
```

**بدنه درخواست:**

```json
{
  "ids": ["room-uuid-1", "room-uuid-2"]
}
```

**پاسخ (202):**

```json
{
  "results": {
    "room-uuid-1": {"success": true, "name": "team-standup"},
    "room-uuid-2": {"success": false, "name": "design-review", "error": "room not found"}
  },
  "totalProcessed": 1,
  "totalFailed": 1
}
```

| فیلد | نوع | توضیحات |
|-------|------|-------------|
| `results` | object | نتیجه به ازای هر اتاق، indexed توسط ID |
| `results.*.success` | bool | آیا عملیات در صف قرار گرفت |
| `results.*.name` | string | نام اتاق (در صورت موفقیت وجود دارد) |
| `results.*.error` | string | پیام خطا (در صورت شکست وجود دارد) |
| `totalProcessed` | int | تعداد اتاق‌های موفق در صف |
| `totalFailed` | int | تعداد اتاق‌های ناموفق |

### بستن گروهی اتاق‌ها

قرار دادن حذف دائمی برای چندین اتاق به طور همزمان در صف. قابل برگشت نیست.

```
POST /api/admin/rooms/bulk/close
```

فرمت درخواست/پاسخ مشابه [تعلیق گروهی اتاق‌ها](#تعلیق-گروهی-اتاق‌ها) است.

به‌روزرسانی تنظیمات سیستم. فیلدهای محرمانه را می‌توان به صورت `"••••••••"` یا رشته خالی ارسال کرد تا مقدار موجود حفظ شود — سرور فقط زمانی مقادیر محرمانه را بازنویسی می‌کند که یک مقدار واقعی ارسال شود.

```
PUT /api/admin/settings
```

**بدنه درخواست (مثال جزئی):**

```json
{
  "registrationEnabled": false,
  "tokenRegistrationOnly": true,
  "passkeysEnabled": true,
  "googleClientId": "your-client-id.apps.googleusercontent.com",
  "googleClientSecret": "GOCSPX-real-secret-value",
  "googleRedirectUrl": "https://your-server.com/api/auth/google/callback",
  "livekitHost": "127.0.0.1:7880",
  "livekitApiKey": "APILiveKitKey123",
  "livekitApiSecret": "real-livekit-secret",
  "logLevel": "debug"
}
```

**پاسخ (200):** شیء کامل تنظیمات را با مقادیر محرمانه پوشانده شده برمی‌گرداند.

پس از ذخیره، سرور به طور خودکار ارائه‌دهندگان OAuth و پیکربندی CORS را reload (بارگذاری مجدد) می‌کند.

> **فیلدهای محرمانه:** `googleClientSecret`، `githubClientSecret`، `twitterClientSecret`، `jwtSecret`، `sessionSecret`، `livekitApiSecret`، `chatUploadS3SecretKey`

## توکن‌های دعوت

توکن‌های دعوت امکان ثبت‌نام محدود را فراهم می‌کنند. وقتی `tokenRegistrationOnly` در تنظیمات فعال باشد، کاربران جدید باید یک توکن دعوت معتبر و استفاده‌نشده در هنگام ثبت‌نام ارائه دهند.

### لیست توکن‌های دعوت

دریافت تمام توکن‌های دعوت، جدیدترین ابتدا. هر توکن شامل یک فیلد `used` بولی است که از `usedAt` محاسبه می‌شود.

```
GET /api/admin/invite-tokens
```

**پاسخ (200):**

```json
{
  "tokens": [
    {
      "id": "token-uuid-1",
      "token": "a1b2c3d4e5f6...",
      "email": "invited@example.com",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-06-15T10:30:00Z",
      "usedAt": null,
      "usedBy": "",
      "createdAt": "2025-03-15T10:30:00Z",
      "used": false
    },
    {
      "id": "token-uuid-2",
      "token": "f6e5d4c3b2a1...",
      "email": "",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-04-15T10:30:00Z",
      "usedAt": "2025-03-20T14:00:00Z",
      "usedBy": "new-user-uuid",
      "createdAt": "2025-03-12T10:30:00Z",
      "used": true
    }
  ]

```

### ایجاد توکن دعوت

ایجاد یک توکن دعوت جدید. مقدار توکن یک رشته هگز تصادفی ۳۲ کاراکتری است.

```
POST /api/admin/invite-tokens
```

**بدنه درخواست:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| فیلد | نوع | ضروری | پیش‌فرض | توضیحات |
|---|---|---|---|---|
| `email` | string | خیر | _ندارد_ | محدود کردن توکن به این آدرس ایمیل |
| `expiresInHours` | int | خیر | 72 | مدت اعتبار توکن به ساعت |

**پاسخ (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

---

### حذف توکن دعوت

ابطال یک توکن دعوت با شناسه آن. پس از حذف، توکن دیگر برای ثبت‌نام قابل استفاده نیست.

```
DELETE /api/admin/invite-tokens/:id
```

**پارامترهای مسیر:**

| پارامتر | توضیحات |
|---|---|
| `id` | UUID توکن دعوت |

**پاسخ (200):**

```json
{
  "status": "success"
}
```

### اطلاعات گواهی

دریافت فراداده گواهی TLS شامل وضعیت انقضا.

```
GET /api/admin/cert-info
```

**پاسخ (200) — TLS فعال، گواهی معتبر:**

```json
{
  "enabled": true,
  "status": "valid",
  "daysRemaining": 78,
  "notAfter": "2025-07-30T12:00:00Z",
  "subject": "CN=bedrud.example.com",
  "issuer": "CN=R3,O=Let's Encrypt,C=US",
  "notBefore": "2025-04-01T12:00:00Z",
  "sans": ["bedrud.example.com", "www.bedrud.example.com"]
}
```

**پاسخ (200) — TLS غیرفعال:**

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| وضعیت | معنی |
|---|---|
| `valid` | گواهی معتبر است و نزدیک به انقضا نیست |
| `expiring_soon` | گواهی ظرف ۳۰ روز آینده منقضی می‌شود |
| `expired` | گواهی منقضی شده است |
| `not_configured` | TLS غیرفعال است |
| `error` | فایل گواهی موجود نیست یا نامعتبر است |

## پاسخ‌های خطا

```json
{
  "error": "forbidden"
}
```

| وضعیت | معنی |
|---|---|
| 400 | ورودی نامعتبر (مثلاً JSON نامعتبر، فیلدهای ضروری موجود نیست، حذف حساب خود، تعلیق اتاق غیرفعال) |
| 401 | احراز هویت نشده — JWT موجود نیست یا منقضی شده |
| 403 | superadmin نیست — کاربر `superadmin` در آرایه `accesses` خود ندارد |
| 404 | منبع یافت نشد (کاربر، اتاق، یا توکن) |
| 409 | تداخل — کاربر نمی‌تواند حساب خود را حذف کند |
| 500 | خطای داخلی سرور |
| 501 | پیاده‌سازی نشده (مثلاً تولید token اتاق `superadmin`) |
| 503 | اطلاعات گواهی در دسترس نیست (خطای TLS) |
