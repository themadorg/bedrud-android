چگونه از اپ وب وارد اتاق Bedrud شوید: پیش‌نمایش پیش‌ازپیوستن، اتصال، کنترل‌های پایه رسانه، و دروازه‌های دسترسی.

## پیش‌ازپیوستن (پوسته خوش‌آمد)

قبل از اتصال رسانه معمولاً می‌بینید:

- **نام** نمایشی
- پیش‌نمایش **میکروفون** و **دوربین**
- **انتخابگر دستگاه** برای میکروفون، دوربین و بلندگو (در صورت موجود بودن)

قبل از پیوستن این‌ها را تنظیم کنید تا اولین اتصال از دستگاه‌های درست استفاده کند.

## پیوستن و اتصال

1. **پیوستن** (یا معادل آن) را بزنید.
2. باید یک پوسته بارگذاری کوتاه ببینید، سپس UI جلسه **متصل** — نه اسپینر بی‌پایان.
3. بقیه شرکت‌کنندگان بعد از پیوستن به‌صورت کاشی یا آواتار ظاهر می‌شوند.

اگر اتصال شکست خورد، مجوزهای مرورگر، شبکه، و در دسترس بودن پورت‌های LiveKit/WebRTC سرور را بررسی کنید. اپراتورها: [اتصال WebRTC](/fa/docs/architecture/webrtc-connectivity).

## کنترل‌های پایه

| عمل | رفتار مورد انتظار |
|-----|-------------------|
| قطع / وصل میکروفون | دیگران دیگر/دوباره صدای شما را می‌شنوند |
| روشن / خاموش دوربین | کاشی ویدیوی شما برای دیگران به‌روز می‌شود |
| ترک اتاق | تمیز قطع می‌شوید؛ دیگران نباید شرکت‌کننده zombie نگه دارند |
| پیوستن مجدد | بعد از ترک، پیوستن دوباره به همان اتاق کار می‌کند |

## چند شرکت‌کننده

با دو مرورگر (یا عادی + خصوصی) تأیید کنید:

- هر دو طرف کاشی یکدیگر را می‌بینند
- وضعیت قطع صدا و دوربین از راه دور دیده می‌شود

## اتاق‌های دربسته

اتاق‌ها ممکن است **رمز** بخواهند، منتظر تأیید بمانند، یا بسته به تنظیمات اتاق بررسی‌های دسترسی دیگری داشته باشند. وقتی **دیالوگ دسترسی اتاق** نشان داده شد کاملش کنید، سپس به پیش‌ازپیوستن / پیوستن بروید.

ورود مهمان (بدون حساب) ممکن است در بعضی نمونه‌ها مجاز باشد — لینک را بفرستید و مهمان فقط با نام نمایشی وارد شود. اگر اتاق پر باشد یا ثبت ظرفیت شکست بخورد، پیوستن ناموفق است (بدون توکن رسانه). مهمان‌ها **کوکی هویت امضاشده پایدار** (`bedrud_guest_id`) می‌گیرند تا مسدودسازی اتاق بین بازدیدها بماند. **تعداد** لابی اتاق‌های عمومی می‌تواند عمومی باشد؛ فهرست هویت شرکت‌کنندگان نیاز به احراز هویت دارد.
