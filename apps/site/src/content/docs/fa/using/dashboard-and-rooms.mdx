بعد از ورود، داشبورد پایگاه اصلی ساخت اتاق، پیوستن به آن‌ها و بازگشت به اتاق‌هایی است که اخیراً استفاده کرده‌اید.

## ساخت اتاق

1. از داشبورد یک اتاق جدید بسازید (نام و هر گزینه اتاقی که میزبان ارائه می‌دهد).
2. اتاق در فهرست شما ظاهر می‌شود.
3. اتاق را باز کنید تا وارد جریان پیش‌ازپیوستن / جلسه شوید.

کارت‌های اتاق از پیوستن و حذف پشتیبانی می‌کنند. تراکم چیدمان ممکن است با نسخه UI فرق کند، اما پیوستن و حذف در دسترس می‌مانند.

## پیوستن سریع

یک **slug اتاق** یا **URL کامل جلسه** را در پیوستن سریع بچسبانید. Bedrud اتاق را resolve می‌کند و شما را به مسیر درست هدایت می‌کند.

لینک‌های قابل اشتراک معمولاً شبیه این‌اند:

```
https://your-host/m/<room-id-or-slug>
```

## اتاق‌های اخیر

بعد از **پیوستن** به اتاق، به **اتاق‌های اخیر** اضافه می‌شود.

- فهرست بعد از **بارگذاری کامل صفحه** باقی می‌ماند.
- یک ورودی اخیر را **حذف** کنید — بعد از بارگذاری مجدد هم حذف‌شده می‌ماند.
- با کارت اخیر سریع همان اتاق را دوباره باز کنید.

اتاق‌های اخیر فهرست راحتی سمت کلاینت است؛ جایگزین فهرست سمت سرور اتاق‌هایی که مالک آن‌اید یا به آن‌ها دسترسی دارید نمی‌شود.

## صفحه فرود

وقتی ریشه نمونه را باز می‌کنید:

- **خارج از حساب** — مسیر بازاریابی / ورود (بدون پروکسی API خراب)
- **واردشده** — ورود به داشبورد
