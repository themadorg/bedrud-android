Bedrud یک تخته سفید مشترک دارد تا همه در اتاق بتوانند همان صحنه را رسم، انتخاب و ویرایش کنند.

## فعال‌سازی قابلیت

تخته سفید می‌تواند پشت **تنظیمات → آزمایشی → تخته سفید مشترک** باشد. روشن کنید، سپس کنترل تخته را در جلسه باز کنید. ممکن است قبل از هر نشست از شما تأیید خواسته شود.

## باز کردن تخته

1. به جلسه بپیوندید (برای تست واقعی دو مرورگر توصیه می‌شود).
2. همپوشانی **تخته سفید** را از کنترل‌های جلسه باز کنید.
3. روی تخته کاربر A رسم کنید — کاربر B باید خطوط را در حدود **یک ثانیه** ببیند.
4. روی B رسم کنید — A باید همان به‌روزرسانی‌ها را ببیند.

## ویرایش مشترک

| عمل | انتظار از راه دور |
|-----|-------------------|
| کشیدن خطوط | سریع برای دیگران ظاهر می‌شود |
| انتخاب / جابه‌جایی / حذف | صحنه راه دور به‌روز می‌شود |
| نشانگر راه دور / دنبال‌کردن | نباید UI را قفل کند |

## بستن، بازکردن، پیوستن مجدد

- همپوشانی را **ببندید** و دوباره باز کنید — صحنه مشترک معمولاً می‌ماند (مگر عمداً پاک شود).
- وسط نشست **ترک** کنید — بدون حلقه مرگ کنسول؛ پیوستن مجدد باید پایدار باشد.
- پیوسته‌های دیرهنگام وضعیت مشترک فعلی را از همان مسیر همگام دریافت می‌کنند.

## چطور کار می‌کند (کوتاه)

تخته روی بوم مشارکتی (ابزار Excalidraw-مانند) با وضعیت CRDT **Yjs** ساخته شده که روی **data channelهای LiveKit** همگام می‌شود. رسانه (صدا/تصویر) روی trackهای WebRTC می‌ماند؛ به‌روزرسانی‌های تخته از مسیرهای data مطمئن/lossy می‌روند تا همکاری به سرور تخته جدا نیاز نداشته باشد.

خودمیزبان‌ها: مطمئن شوید build تولید شامل vendor chunk تخته باشد (`bun run build` برای اپ وب) تا باز کردن تخته روی assetهای استاتیک 404 ندهد.
