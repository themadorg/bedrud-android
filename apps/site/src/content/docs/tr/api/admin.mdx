Yönetici uç noktaları `/api/admin/` altındadır. Tümü **superadmin** erişim seviyesi ve `Authorization: Bearer <token>` başlığında geçerli bir JWT gerektirir.

## Genel Bakış

| Uç Nokta | Yöntem | Açıklama |
|---|---|---|
| `/api/admin/users` | GET | Tüm kullanıcıları listele (sayfalandırmalı) |
| `/api/admin/users/:id` | GET | Kullanıcı detayını odalarla birlikte getir |
| `/api/admin/users/:id/status` | PUT | Bir kullanıcıyı aktifleştir veya devre dışı bırak |
| `/api/admin/users/:id/accesses` | PUT | Bir kullanıcının erişim seviyelerini güncelle |
| `/api/admin/users/:id` | DELETE | Bir kullanıcıyı kalıcı olarak sil |
| `/api/admin/rooms` | GET | Tüm odaları listele (sayfalandırmalı) |
| `/api/admin/rooms/:roomId` | PUT | Herhangi bir odanın ayarlarını güncelle |
| `/api/admin/rooms/:roomId` | DELETE | Bir odayı kapat (kademeli silme) |
| `/api/admin/rooms/:roomId/suspend` | POST | Aktif bir odayı askıya al |
| `/api/admin/rooms/:roomId/participants` | GET | LiveKit'ten canlı katılımcı verilerini al |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | Herhangi bir katılımcıyı odadan at |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | Herhangi bir katılımcının sesini kapat |
| `/api/admin/settings` | GET | Sistem ayarlarını getir (gizli anahtarlar maskelenmiş) |
| `/api/admin/settings` | PUT | Sistem ayarlarını güncelle |
| `/api/admin/invite-tokens` | GET | Tüm davet jetonlarını listele |
| `/api/admin/invite-tokens` | POST | Yeni bir davet jetonu oluştur |
| `/api/admin/invite-tokens/:id` | DELETE | Bir davet jetonunu sil |
| `/api/admin/online-count` | GET | Toplam aktif katılımcı sayısı |
| `/api/admin/livekit/stats` | GET | Toplu LiveKit sunucu istatistikleri |
| `/api/admin/cert-info` | GET | TLS sertifika meta verileri |

## Erişim Seviyeleri

Bedrud çok seviyeli bir erişim sistemi kullanır. Her kullanıcının aynı anda birden fazla seviye içerebilen bir `accesses` dizisi vardır.

| Erişim Seviyesi | Açıklama | Middleware Koruması |
|---|---|---|
| `superadmin` | Tam sistem erişimi. Yönetici paneli, kullanıcı/oda yönetimi, ayarlar, davet jetonları. | `/api/admin/*` üzerinde `RequireAccess("superadmin")` |
| `admin` | Tanımlı ancak middleware ile zorlanmıyor. Yalnızca bilgilendirme amaçlı. | Şu anda yok |
| `moderator` | Oda seviyesinde moderasyon. Sohbeti engelleme, sağırlaştırma, spot ışığı, oda başına videoyu devre dışı bırakma. | Oda kapsamlı `isRoomModerator()` kontrolü |
| `user` | Standart kimliği doğrulanmış kullanıcı. Oda oluşturma/katılma, profili yönetme. | Kullanıcı rotalarında `Protected()` |
| `guest` | Genel oda katılımları için geçici kullanıcı. Yalnızca misafir girişi. | Yok |

> **Not:** `/api/admin/*` rota grubu `admin` değil, `superadmin` kontrolü yapar. Yalnızca `accesses` dizisinde `superadmin` bulunan kullanıcılar bu uç noktaları kullanabilir.

## Kullanıcı Yönetimi

### Tüm Kullanıcıları Listele

Tüm kayıtlı kullanıcıların sayfalandırılmış listesini getir.

```
GET /api/admin/users?page=1&limit=50
```

**Başlıklar:** `Authorization: Bearer <accessToken>`

**Sorgu Parametreleri:**

| Parametre | Tür | Varsayılan | Açıklama |
|---|---|---|---|
| `page` | int | 1 | Sayfa numarası |
| `limit` | int | 50 | Sayfa başına kullanıcı |

**Yanıt (200):**

```json
{
  "users": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",
      "name": "John Doe",
      "provider": "local",
      "isActive": true,
      "isAdmin": false,
      "accesses": ["user"],
      "createdAt": "2025-01-15 10:30:00"
    },
    {
      "id": "123e4567-e89b-12d3-a456-426614174001",
      "email": "admin@example.com",
      "name": "Admin User",
      "provider": "local",
      "isActive": true,
      "isAdmin": true,
      "accesses": ["user", "superadmin"],
      "createdAt": "2025-01-10 08:00:00"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

### Kullanıcı Durumunu Güncelle

Bir kullanıcı hesabını aktifleştir veya devre dışı bırak. Devre dışı bırakılan kullanıcılar giriş yapamaz veya jeton yenileyemez.

```
PUT /api/admin/users/:id/status
```

**Yol Parametreleri:**

| Parametre | Açıklama |
|---|---|
| `id` | Kullanıcı UUID'si |

**İstek Gövdesi:**

```json
{
  "active": false
}
```

**Yanıt (200):**

```json
{
  "message": "User status updated successfully"
}
```

---

### Kullanıcı Erişimlerini Güncelle

Bir kullanıcının erişim seviyesi dizisini değiştir. superadmin'e yükseltmek, düşürmek veya moderator erişimi eklemek için kullanın.

```
PUT /api/admin/users/:id/accesses
```

**Yol Parametreleri:**

| Parametre | Açıklama |
|---|---|
| `id` | Kullanıcı UUID'si |

**İstek Gövdesi:**

```json
{
  "accesses": ["user", "superadmin"]
}
```

| Alan | Tür | Açıklama |
|---|---|---|
| `accesses` | string[] | Erişim seviyeleri dizisi. Geçerli değerler: `user`, `superadmin`, `admin`, `moderator`, `guest` |

**Yanıt (200):**

```json
{
  "message": "User accesses updated"
}
```

---

### Kullanıcı Sil

Bir kullanıcıyı ve ilişkili tüm verileri kalıcı olarak sil. Asenkron çalışır — hemen `202 Accepted` döndürür. Kendi hesabınızı silemezsiniz.

```
DELETE /api/admin/users/:id
```

**Yol Parametreleri:**

| Parametre | Açıklama |
|---|---|
| `id` | Kullanıcı UUID'si |

**Yanıt (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| Alan | Açıklama |
|---|---|
| `message` | Silme işleminin devam ettiğine dair onay |
| `rooms` | Bu kullanıcı tarafından oluşturulan oda sayısı (bunlar da silinir) |

Arka plan görevi şunları kademeli olarak siler:
- Kullanıcı tarafından oluşturulan tüm odalar
- İlişkili passkey'ler (WebAuthn kimlik bilgileri)
- Kullanıcı tercihleri
- Kullanıcı kaydının kendisi

### Odayı Güncelle

Yalnızca admin'in kullanabileceği `isPersistent` bayrağı dahil herhangi bir odanın ayarlarını değiştir.

```
PUT /api/admin/rooms/:roomId
```

**Yol Parametreleri:**

| Parametre | Açıklama |
|---|---|
| `roomId` | Oda UUID'si |

**İstek Gövdesi:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| Alan | Tür | Açıklama |
|---|---|---|
| `maxParticipants` | int | İzin verilen maksimum katılımcı (isteğe bağlı) |
| `settings.allowChat` | bool | Oda içi sohbeti etkinleştir |
| `settings.allowAudio` | bool | Ses izlerini etkinleştir |
| `settings.allowVideo` | bool | Video izlerini etkinleştir |
| `settings.requireApproval` | bool | Katılım için admin onayı gerektir |
| `settings.e2ee` | bool | Uçtan uca şifrelemeyi etkinleştir |
| `settings.isPersistent` | bool | **Yalnızca admin.** Boşta temizlemeyi engelle |

Tüm ayar alanları isteğe bağlıdır — yalnızca değiştirmek istediklerinizi ekleyin.

**Yanıt (200):** Güncellenmiş oda nesnesini döndürür.

---

### Odayı Kapat

Bir odayı kademeli silme ile kalıcı olarak kapat. Tüm katılımcılara bir sistem mesajı gönderir: "This room has been closed by an administrator".

```
DELETE /api/admin/rooms/:roomId
```

**Yol Parametreleri:**

| Parametre | Açıklama |
|---|---|
| `roomId` | Oda UUID'si |

**Yanıt (200):**

```json
{
  "status": "success"
}
```

---

### Odayı Askıya Al

Oda kaydını silmeden aktif bir odayı ve tüm katılımcılarını devre dışı bırak. Bir katılımcı tekrar katıldığında oda yeniden aktifleştirilebilir.

```
POST /api/admin/rooms/:roomId/suspend
```

**Yol Parametreleri:**

| Parametre | Açıklama |
|---|---|
| `roomId` | Oda UUID'si |

**Yanıt (200):**

```json
{
  "status": "success"
}
```

**Hata (400):** Oda zaten aktif değilse `{"error": "Room is not active"}` döndürür.

```
PUT /api/admin/settings
```

**İstek Gövdesi (kısmi örnek):**

```json
{

  "tokenRegistrationOnly": true,
  "passkeysEnabled": true,
```

## Davet Jetonları

Davet jetonları kontrollü kaydı etkinleştirir. Ayarlarda `tokenRegistrationOnly` etkin olduğunda, yeni kullanıcılar kayıt sırasında geçerli, kullanılmamış bir davet jetonu sağlamalıdır.

Tüm davet jetonlarını en yeniden eskiye getir. Her jeton, `usedAt` alanından hesaplanan bir `used` boolean değeri içerir.

```
GET /api/admin/invite-tokens
```

```json
      "token": "a1b2c3d4e5f6...",

      "usedAt": null,
      "usedBy": "",
      "createdAt": "2025-03-15T10:30:00Z",

    {
      "id": "token-uuid-2",
      "token": "f6e5d4c3b2a1...",
```

### Davet Jetonu Oluştur

Yeni bir davet jetonu oluştur. Jeton değeri 32 karakterlik rastgele bir hex string'tir.

```
POST /api/admin/invite-tokens
```

**İstek Gövdesi:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| Alan | Tür | Zorunlu | Varsayılan | Açıklama |
|---|---|---|---|---|
| `email` | string | Hayır | _yok_ | Jetonu bu e-posta adresiyle sınırla |
| `expiresInHours` | int | Hayır | 72 | Saat cinsinden jeton geçerlilik süresi |

**Yanıt (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

### Davet Jetonunu Sil

**Yol Parametreleri:**

| Parametre | Açıklama |
|---|---|
| `id` | Davet jetonu UUID'si |

**Yanıt (200):**

```json

  "status": "success"
}
```

### Sertifika Bilgisi

Sona erme durumu dahil TLS sertifika meta verilerini getir.

```
GET /api/admin/cert-info
```

**Yanıt (200) — TLS etkin, sertifika geçerli:**

```json
{
  "enabled": true,
  "status": "valid",
  "daysRemaining": 78,
  "notAfter": "2025-07-30T12:00:00Z",
  "subject": "CN=bedrud.example.com",
  "issuer": "CN=R3,O=Let's Encrypt,C=US",
  "notBefore": "2025-04-01T12:00:00Z",
  "sans": ["bedrud.example.com", "www.bedrud.example.com"]
}
```

**Yanıt (200) — TLS etkin değil:**

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| Durum | Anlamı |
|---|---|
| `valid` | Sertifika geçerli ve sona ermeye yakın değil |
| `expiring_soon` | Sertifika 30 gün içinde sona eriyor |
| `expired` | Sertifikanın süresi dolmuş |
| `not_configured` | TLS devre dışı |
| `error` | Sertifika dosyası eksik veya geçersiz |

## Hata Yanıtları

```json
{
  "error": "forbidden"
}
```

| Durum | Anlamı |
|---|---|
| 400 | Geçersiz girdi (ör. bozuk JSON, eksik zorunlu alanlar, kendi hesabını silme, aktif olmayan odayı askıya alma) |
| 401 | Kimlik doğrulaması yapılmamış — JWT eksik veya süresi dolmuş |
| 403 | superadmin değil — kullanıcının `accesses` dizisinde `superadmin` yok |
| 404 | Kaynak bulunamadı (kullanıcı, oda veya jeton) |
| 409 | Çakışma — kullanıcı kendi hesabını silemez |
| 500 | Dahili sunucu hatası |
| 501 | Uygulanmadı (ör. admin oda jetonu oluşturma) |
| 503 | Sertifika bilgisi kullanılamıyor (TLS hatası) |
