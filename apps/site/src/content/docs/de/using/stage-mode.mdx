Der Stage-Modus wechselt das Raum-Layout so, dass eine präsentierende Person (oder ein Stage-Set) im Vordergrund steht — statt des normalen Grid- oder Spotlight-Layouts.

## Stage starten

1. Als Host (oder Rolle mit Stage-Kontrolle) **starten Sie die Stage** über die Meeting-Steuerung.
2. Andere Teilnehmer sehen das **Stage-Layout** und ggf. den **Beitritts-Hinweis** für Stage-Sitzungen.
3. Die Teilnehmerleiste / Seitenleiste kann den Rest des Raums weiterhin anzeigen (umschaltbar, falls vorhanden).

## Bildschirmfreigabe auf der Stage

- Teilen Sie den Bildschirm bei aktiver Stage — andere sehen ein **Stage-Bildschirmfreigabe-Overlay**.
- **Freigabe beenden** entfernt das Overlay.

Zur Bildschirmfreigabe ohne Stage (normales Grid) siehe [Meeting-Steuerung](/de/docs/using/meeting-controls).

## Stage beenden

**Stage beenden** bringt den Raum zurück ins normale Grid- oder Spotlight-Layout.

## Späte Beitretende

Wer **bei bereits aktiver Stage** beitritt, sollte den korrekten Stage-Zustand sofort sehen — nicht dauerhaft ein leeres Grid. Sieht ein Spätkommender das falsche Layout, einmal neu laden; bleibt es falsch, als Regression melden.

## Tipps

- Stage eignet sich für Demos, Lehre und Präsentationen an viele.
- Kombinieren Sie mit [YouTube Watch](/de/docs/using/youtube-watch) oder [Whiteboard](/de/docs/using/whiteboard) nur, wenn diese Overlays zusammenpassen; schalten Sie sorgfältig, damit Teilnehmende nicht unter mehreren Vollbildschichten stecken bleiben.
