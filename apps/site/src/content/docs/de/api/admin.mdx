Admin-Endpunkte befinden sich unter `/api/admin/`. Alle erfordern die Zugriffsebene **superadmin** und ein gültiges JWT im `Authorization: Bearer <token>`-Header.

## Übersicht

| Endpunkt | Methode | Beschreibung |
|---|---|---|
| `/api/admin/users` | GET | Alle Benutzer auflisten (paginiert) |
| `/api/admin/users/:id` | GET | Benutzerdetails mit Räumen abrufen |
| `/api/admin/users/:id/status` | PUT | Benutzer aktivieren oder deaktivieren |
| `/api/admin/users/:id/accesses` | PUT | Zugriffsebenen eines Benutzers aktualisieren |
| `/api/admin/users/:id` | DELETE | Benutzer dauerhaft löschen |
| `/api/admin/rooms` | GET | Alle Räume auflisten (paginiert) |
| `/api/admin/rooms/:roomId` | PUT | Raumeinstellungen bearbeiten |
| `/api/admin/rooms/:roomId` | DELETE | Raum löschen (Hintergrundjob) |
| `/api/admin/rooms/:roomId/suspend` | POST | Raum aussetzen (Hintergrundjob) |
| `/api/admin/rooms/bulk/suspend` | POST | Räume massenhaft aussetzen |
| `/api/admin/rooms/bulk/close` | POST | Räume massenhaft schließen |
| `/api/admin/users/bulk/delete` | POST | Benutzer massenhaft löschen |
| `/api/admin/rooms/:roomId/participants` | GET | Live-Teilnehmerdaten von LiveKit abrufen |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | Teilnehmer aus einem Raum entfernen |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | Audio eines Teilnehmers stummschalten |
| `/api/admin/settings` | GET | Systemeinstellungen abrufen (Secrets maskiert) |
| `/api/admin/settings` | PUT | Systemeinstellungen aktualisieren |
| `/api/admin/invite-tokens` | GET | Alle Einladungstokens auflisten |
| `/api/admin/invite-tokens` | POST | Neuen Einladungstoken erstellen |
| `/api/admin/invite-tokens/:id` | DELETE | Einladungstoken löschen |
| `/api/admin/online-count` | GET | Gesamtzahl aktiver Teilnehmer |
| `/api/admin/livekit/stats` | GET | Aggregierte LiveKit-Serverstatistiken |
| `/api/admin/cert-info` | GET | TLS-Zertifikat-Metadaten |

## Zugriffsebenen

Bedrud verwendet ein mehrstufiges Zugriffssystem. Jeder Benutzer verfügt über ein `accesses`-Array, das mehrere Ebenen gleichzeitig enthalten kann.

| Zugriffsebene | Beschreibung | Middleware-Schutz |
|---|---|---|
| `superadmin` | Voller Systemzugriff. Admin-Panel, Benutzer-/Raumverwaltung, Einstellungen, Einladungstokens. | `RequireAccess("superadmin")` auf `/api/admin/*` |
| `admin` | Definiert, aber nicht durch Middleware erzwungen. Nur informativ. | Keine (derzeit) |
| `moderator` | Raumbezogene Moderation. Chat blockieren, stummschalten, Spotlight, Video deaktivieren – pro Raum. | Raumbezogene `isRoomModerator()`-Prüfung |
| `user` | Standardmäßig authentifizierter Benutzer. Räume erstellen/betreten, Profil verwalten. | `Protected()` auf Benutzerrouten |
| `guest` | Temporärer Benutzer für öffentliche Raumbeitritte. Nur Gastanmeldung. | N/A |

> **Hinweis:** Die `/api/admin/*`-Routengruppe prüft auf `superadmin`, nicht auf `admin`. Nur Benutzer mit `superadmin` in ihrem `accesses`-Array können diese Endpunkte nutzen.

## Benutzerverwaltung

### Alle Benutzer auflisten

Eine paginierte Liste aller registrierten Benutzer abrufen.

```
GET /api/admin/users?page=1&limit=50
```

**Header:** `Authorization: Bearer <accessToken>`

**Query-Parameter:**

| Parameter | Typ | Standard | Beschreibung |
|---|---|---|---|
| `page` | int | 1 | Seitenzahl |
| `limit` | int | 50 | Benutzer pro Seite |

**Antwort (200):**

```json
{
  "users": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",
      "name": "John Doe",
      "provider": "local",
      "isActive": true,
      "isAdmin": false,
      "accesses": ["user"],
      "createdAt": "2025-01-15 10:30:00"
    },
    {
      "id": "123e4567-e89b-12d3-a456-426614174001",
      "email": "admin@example.com",
      "name": "Admin User",
      "provider": "local",
      "isActive": true,
      "isAdmin": true,
      "accesses": ["user", "superadmin"],
      "createdAt": "2025-01-10 08:00:00"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

### Benutzerstatus aktualisieren

Ein Benutzerkonto aktivieren oder deaktivieren. Deaktivierte Benutzer können sich nicht anmelden oder Tokens aktualisieren.

```
PUT /api/admin/users/:id/status
```

**Pfadparameter:**

| Parameter | Beschreibung |
|---|---|
| `id` | Benutzer-UUID |

**Anfrage-Body:**

```json

  "active": false
}
```

**Antwort (200):**

```json
{
  "message": "User status updated successfully"
}
```

Das Zugriffsebenen-Array eines Benutzers ändern. Verwenden Sie dies, um zum Superadmin zu befördern, herabzustufen oder Moderator-Zugriff hinzuzufügen.

```
PUT /api/admin/users/:id/accesses
```

**Pfadparameter:**

| Parameter | Beschreibung |
|---|---|
| `id` | Benutzer-UUID |

**Anfrage-Body:**

```json
{
  "accesses": ["user", "superadmin"]
}
```

| Feld | Typ | Beschreibung |
|---|---|---|
| `accesses` | string[] | Array von Zugriffsebenen. Gültige Werte: `user`, `superadmin`, `admin`, `moderator`, `guest` |

**Antwort (200):**

```json
{
  "message": "User accesses updated"
}
```

### Benutzer löschen

Einen Benutzer und alle zugehörigen Daten dauerhaft löschen. Läuft asynchron – gibt sofort `202 Accepted` zurück. Das eigene Konto kann nicht gelöscht werden.

```
DELETE /api/admin/users/:id
```

**Pfadparameter:**

| Parameter | Beschreibung |
|---|---|
| `id` | Benutzer-UUID |

**Antwort (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| Feld | Beschreibung |
|---|---|
| `message` | Bestätigung, dass die Löschung läuft |
| `rooms` | Anzahl der vom Benutzer erstellten Räume (werden ebenfalls gelöscht) |

Der Hintergrundjob führt eine Kaskadenlöschung durch für:
- Alle vom Benutzer erstellten Räume
- Zugehörige Passkeys (WebAuthn-Anmeldedaten)
- Benutzereinstellungen
- Den Benutzerdatensatz selbst

### Raum bearbeiten

Die Einstellungen eines beliebigen Raums ändern, einschließlich des Admin-exklusiven `isPersistent`-Flags.

```
PUT /api/admin/rooms/:roomId
```

**Pfadparameter:**

| Parameter | Beschreibung |
|---|---|
| `roomId` | Raum-UUID |

**Anfrage-Body:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| Feld | Typ | Beschreibung |
|---|---|---|
| `maxParticipants` | int | Maximal zugelassene Teilnehmer (optional) |
| `settings.allowChat` | bool | In-Raum-Chat aktivieren |
| `settings.allowAudio` | bool | Audiospuren aktivieren |
| `settings.allowVideo` | bool | Videospuren aktivieren |
| `settings.requireApproval` | bool | Admin-Genehmigung für Beitritt erforderlich |
| `settings.e2ee` | bool | Ende-zu-Ende-Verschlüsselung aktivieren |
| `settings.isPersistent` | bool | **Nur Admin.** Automatische Bereinigung inaktiver Räume verhindern |

Alle Einstellungsfelder sind optional – geben Sie nur die Felder an, die Sie ändern möchten.

**Antwort (200):** Gibt das aktualisierte Raumobjekt zurück.

---

### Raum schließen

Einen Raum dauerhaft löschen. Der Löschvorgang wird als Hintergrundjob ausgeführt und kehrt sofort zurück — der Raum wird innerhalb weniger Sekunden vollständig gelöscht.

```
DELETE /api/admin/rooms/:roomId
```

**Pfadparameter:**

| Parameter | Beschreibung |
|---|---|
| `roomId` | Raum-UUID |

**Antwort (202):**

```json
{
  "message": "Room deletion queued"
}
```

| Status | Beschreibung |
|--------|-------------|
| 202 | Löschung vorgemerkt — wird in Kürze abgeschlossen |
| 404 | Raum nicht gefunden |
| 500 | Raum konnte nicht abgerufen oder Job nicht eingeplant werden |

> **Hinweis:** Beim Schließen wird eine Systemnachricht an alle aktiven Teilnehmer gesendet: „This room has been closed by an administrator".
> Die Schließlogik ruft `cleanupSvc.CascadeDeleteRoom` auf, das die LiveKit-Raumentfernung, Chat-Upload-Bereinigung
> und das Löschen des Datenbankeintrags übernimmt.

---

### Raum aussetzen

Einen aktiven Raum deaktivieren und alle aktiven Anrufe beenden, ohne den Raumdatensatz zu löschen. Der Raum kann wieder aktiviert werden, wenn ein Teilnehmer erneut beitritt. Wird als Hintergrundjob ausgeführt und kehrt sofort zurück.

```
POST /api/admin/rooms/:roomId/suspend
```

**Pfadparameter:**

| Parameter | Beschreibung |
|---|---|
| `roomId` | Raum-UUID |

**Antwort (202):**

```json
{
  "message": "Room suspend queued"
}
```

| Status | Beschreibung |
|--------|-------------|
| 202 | Aussetzung vorgemerkt — wird in Kürze abgeschlossen |
| 400 | Raum bereits inaktiv |
| 404 | Raum nicht gefunden |
| 500 | Raum konnte nicht abgerufen oder Job nicht eingeplant werden |

---

### Räume massenhaft aussetzen

Aussetzung für mehrere Räume gleichzeitig einplanen. Bereits ausgesetzte Räume werden übersprungen.

```
POST /api/admin/rooms/bulk/suspend
```

**Anfrage-Body:**

```json
{
  "ids": ["room-uuid-1", "room-uuid-2"]
}
```

**Antwort (202):**

```json
{
  "results": {
    "room-uuid-1": {"success": true, "name": "team-standup"},
    "room-uuid-2": {"success": false, "name": "design-review", "error": "room not found"}
  },
  "totalProcessed": 1,
  "totalFailed": 1
}
```

| Feld | Typ | Beschreibung |
|-------|------|-------------|
| `results` | object | Pro-Raum-Ergebnis, nach ID gruppiert |
| `results.*.success` | bool | Ob der Vorgang eingeplant wurde |
| `results.*.name` | string | Raumname (bei Erfolg vorhanden) |
| `results.*.error` | string | Fehlermeldung (bei Fehler vorhanden) |
| `totalProcessed` | int | Anzahl erfolgreich eingeplanter Räume |
| `totalFailed` | int | Anzahl fehlgeschlagener Räume |

### Räume massenhaft schließen

Dauerhafte Löschung für mehrere Räume gleichzeitig einplanen. Kann nicht rückgängig gemacht werden.

```
POST /api/admin/rooms/bulk/close
```

Gleiches Anfrage-/Antwortformat wie [Räume massenhaft aussetzen](#räume-massenhaft-aussetzen).

### Einstellungen aktualisieren

Systemeinstellungen aktualisieren. Geheime Felder können als `"••••••••"` oder leerer String gesendet werden, um den bestehenden Wert zu erhalten – der Server überschreibt Secrets nur, wenn ein echter Wert gesendet wird.

```
PUT /api/admin/settings
```

**Anfrage-Body (Teilbeispiel):**

```json
{
  "registrationEnabled": false,
  "tokenRegistrationOnly": true,
  "passkeysEnabled": true,
  "googleClientId": "your-client-id.apps.googleusercontent.com",
  "googleClientSecret": "GOCSPX-real-secret-value",
  "googleRedirectUrl": "https://your-server.com/api/auth/google/callback",
  "livekitHost": "127.0.0.1:7880",
  "livekitApiKey": "APILiveKitKey123",
  "livekitApiSecret": "real-livekit-secret",
  "logLevel": "debug"
}
```

**Antwort (200):** Gibt das vollständige Einstellungsobjekt mit maskierten Secrets zurück.

Beim Speichern lädt der Server automatisch die OAuth-Provider und die CORS-Konfiguration neu.

> **Geheime Felder:** `googleClientSecret`, `githubClientSecret`, `twitterClientSecret`, `jwtSecret`, `sessionSecret`, `livekitApiSecret`, `chatUploadS3SecretKey`

## Einladungstokens

Einladungstokens ermöglichen eine geschützte Registrierung. Wenn `tokenRegistrationOnly` in den Einstellungen aktiviert ist, müssen neue Benutzer bei der Registrierung einen gültigen, unbenutzten Einladungstoken angeben.

### Einladungstokens auflisten

Alle Einladungstokens abrufen, neueste zuerst. Jeder Token enthält ein `used`-Boolean, das aus `usedAt` berechnet wird.

```
GET /api/admin/invite-tokens
```

**Antwort (200):**

```json
{
  "tokens": [
    {
      "id": "token-uuid-1",
      "token": "a1b2c3d4e5f6...",
      "email": "invited@example.com",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-06-15T10:30:00Z",
      "usedAt": null,
      "usedBy": "",
      "createdAt": "2025-03-15T10:30:00Z",
      "used": false
    },
    {
      "id": "token-uuid-2",
      "token": "f6e5d4c3b2a1...",
      "email": "",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-04-15T10:30:00Z",
      "usedAt": "2025-03-20T14:00:00Z",
      "usedBy": "new-user-uuid",
      "createdAt": "2025-03-12T10:30:00Z",
      "used": true
    }
  ]
}
```

---

### Einladungstoken erstellen

Einen neuen Einladungstoken generieren. Der Token-Wert ist ein 32-stelliger zufälliger Hex-String.

```
POST /api/admin/invite-tokens
```

**Anfrage-Body:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| Feld | Typ | Erforderlich | Standard | Beschreibung |
|---|---|---|---|---|
| `email` | string | Nein | _keiner_ | Token auf diese E-Mail-Adresse beschränken |
| `expiresInHours` | int | Nein | 72 | Gültigkeitsdauer des Tokens in Stunden |

**Antwort (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

---

### Einladungstoken löschen

Einen Einladungstoken anhand seiner ID widerrufen. Nach der Löschung kann der Token nicht mehr zur Registrierung verwendet werden.

```
DELETE /api/admin/invite-tokens/:id
```

**Pfadparameter:**

| Parameter | Beschreibung |
|---|---|
| `id` | Einladungstoken-UUID |

**Antwort (200):**

```json
{
  "status": "success"
}
```

### Zertifikatsinformationen

TLS-Zertifikat-Metadaten einschließlich Ablaufstatus abrufen.

```
GET /api/admin/cert-info
```

**Antwort (200) — TLS aktiviert, Zertifikat gültig:**

```json
{
  "enabled": true,
  "status": "valid",
  "daysRemaining": 78,
  "notAfter": "2025-07-30T12:00:00Z",
  "subject": "CN=bedrud.example.com",
  "issuer": "CN=R3,O=Let's Encrypt,C=US",
  "notBefore": "2025-04-01T12:00:00Z",
  "sans": ["bedrud.example.com", "www.bedrud.example.com"]
}
```

**Antwort (200) — TLS nicht aktiviert:**

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| Status | Bedeutung |
|---|---|
| `valid` | Zertifikat ist gültig und läuft nicht bald ab |
| `expiring_soon` | Zertifikat läuft innerhalb von 30 Tagen ab |
| `expired` | Zertifikat ist abgelaufen |
| `not_configured` | TLS ist deaktiviert |
| `error` | Zertifikatsdatei fehlt oder ist ungültig |

---

## Fehlerantworten

```json
{
  "error": "forbidden"
}
```

| Status | Bedeutung |
|---|---|
| 400 | Ungültige Eingabe (z. B. fehlerhaftes JSON, fehlende Pflichtfelder, Löschen des eigenen Kontos, Aussetzen eines inaktiven Raums) |
| 401 | Nicht authentifiziert – fehlendes oder abgelaufenes JWT |
| 403 | Kein Superadmin – Benutzer hat kein `superadmin` in seinem `accesses`-Array |
| 404 | Ressource nicht gefunden (Benutzer, Raum oder Token) |
| 409 | Konflikt – Benutzer kann sein eigenes Konto nicht löschen |
| 500 | Interner Serverfehler |
| 501 | Nicht implementiert (z. B. Admin-Raum-Token-Generierung) |
| 503 | Zertifikatsinformationen nicht verfügbar (TLS-Fehler) |
