Los endpoints de administración están bajo `/api/admin/`. Todos requieren el nivel de acceso **superadmin** y un JWT válido en el encabezado `Authorization: Bearer <token>`.

## Resumen

| Endpoint | Método | Descripción |
|---|---|---|
| `/api/admin/users` | GET | Listar todos los usuarios (paginado) |
| `/api/admin/users/:id` | GET | Obtener detalle de usuario con sus salas |
| `/api/admin/users/:id/status` | PUT | Activar o desactivar un usuario |
| `/api/admin/users/:id/accesses` | PUT | Actualizar los niveles de acceso de un usuario |
| `/api/admin/users/:id` | DELETE | Eliminar permanentemente un usuario |
| `/api/admin/rooms` | GET | Listar todas las salas (paginado) |
| `/api/admin/rooms/:roomId` | PUT | Actualizar la configuración de cualquier sala |
| `/api/admin/rooms/:roomId` | DELETE | Eliminar una sala (tarea en segundo plano) |
| `/api/admin/rooms/:roomId/suspend` | POST | Suspender una sala (tarea en segundo plano) |
| `/api/admin/rooms/bulk/suspend` | POST | Suspender varias salas |
| `/api/admin/rooms/bulk/close` | POST | Cerrar varias salas |
| `/api/admin/users/bulk/delete` | POST | Eliminar varios usuarios |
| `/api/admin/rooms/:roomId/participants` | GET | Obtener datos de participantes en vivo desde LiveKit |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | Expulsar a cualquier participante de una sala |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | Silenciar el audio de cualquier participante |
| `/api/admin/settings` | GET | Obtener configuración del sistema (secretos enmascarados) |
| `/api/admin/settings` | PUT | Actualizar configuración del sistema |
| `/api/admin/invite-tokens` | GET | Listar todos los tokens de invitación |
| `/api/admin/invite-tokens` | POST | Crear un nuevo token de invitación |
| `/api/admin/invite-tokens/:id` | DELETE | Eliminar un token de invitación |
| `/api/admin/online-count` | GET | Total de participantes activos |
| `/api/admin/livekit/stats` | GET | Estadísticas agregadas del servidor LiveKit |
| `/api/admin/cert-info` | GET | Metadatos del certificado TLS |

## Niveles de Acceso

Bedrud utiliza un sistema de acceso multinivel. Cada usuario tiene un arreglo `accesses` que puede contener múltiples niveles simultáneamente.

| Nivel de Acceso | Descripción | Protección del Middleware |
|---|---|---|
| `superadmin` | Acceso total al sistema. Panel de administración, gestión de usuarios/salas, configuración, tokens de invitación. | `RequireAccess("superadmin")` en `/api/admin/*` |
| `admin` | Definido pero no aplicado por middleware. Solo informativo. | Ninguno actualmente |
| `moderator` | Moderación a nivel de sala. Bloquear chat, ensordecer, destacar, desactivar video por sala. | Verificación `isRoomModerator()` con alcance de sala |
| `user` | Usuario autenticado estándar. Crear/unirse a salas, gestionar perfil. | `Protected()` en rutas de usuario |
| `guest` | Usuario temporal para unirse a salas públicas. Solo inicio de sesión como invitado. | N/A |

> **Nota:** El grupo de rutas `/api/admin/*` verifica `superadmin`, no `admin`. Solo los usuarios con `superadmin` en su arreglo `accesses` pueden usar estos endpoints.

## Gestión de Usuarios

### Listar Todos los Usuarios

Recupera una lista paginada de todos los usuarios registrados.

```
GET /api/admin/users?page=1&limit=50
```

**Encabezados:** `Authorization: Bearer <accessToken>`

**Parámetros de Consulta:**

| Parámetro | Tipo | Valor predeterminado | Descripción |
|---|---|---|---|
| `page` | int | 1 | Número de página |
| `limit` | int | 50 | Usuarios por página |

**Respuesta (200):**

```json
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",

      "isAdmin": false,
```

### Actualizar Estado del Usuario

|---|---|
| `id` | UUID del usuario |


```json

  "message": "User status updated successfully"
}
```

### Eliminar Usuario

Elimina permanentemente un usuario y todos sus datos asociados. Se ejecuta de forma asíncrona — devuelve `202 Accepted` inmediatamente. No puede eliminar su propia cuenta.

```
DELETE /api/admin/users/:id
```

**Parámetros de Ruta:**

| Parámetro | Descripción |
|---|---|
| `id` | UUID del usuario |

**Respuesta (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| Campo | Descripción |
|---|---|
| `message` | Confirmación de que la eliminación está en progreso |
| `rooms` | Número de salas creadas por este usuario (también eliminadas) |

El trabajo en segundo plano elimina en cascada:
- Todas las salas creadas por el usuario
- Las passkeys asociadas (credenciales WebAuthn)
- Las preferencias del usuario
- El registro del usuario en sí

### Actualizar Sala

Modifica la configuración de cualquier sala, incluyendo el flag `isPersistent` exclusivo para administradores.

```
PUT /api/admin/rooms/:roomId
```

**Parámetros de Ruta:**

| Parámetro | Descripción |
|---|---|
| `roomId` | UUID de la sala |

**Cuerpo de la Solicitud:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| Campo | Tipo | Descripción |
|---|---|---|
| `maxParticipants` | int | Máximo de participantes permitidos (opcional) |
| `settings.allowChat` | bool | Habilitar chat en la sala |
| `settings.allowAudio` | bool | Habilitar pistas de audio |
| `settings.allowVideo` | bool | Habilitar pistas de video |
| `settings.requireApproval` | bool | Requerir aprobación del admin para unirse |
| `settings.e2ee` | bool | Habilitar cifrado de extremo a extremo |
| `settings.isPersistent` | bool | **Solo admin.** Evitar la limpieza automática de salas inactivas |

Todos los campos de configuración son opcionales — incluya solo los que desee cambiar.

**Respuesta (200):** Devuelve el objeto de sala actualizado.

---

### Cerrar Sala

Eliminar permanentemente una sala y todos sus participantes. Se encola una tarea en segundo plano y regresa inmediatamente — la sala se eliminará por completo en cuestión de segundos.

```
DELETE /api/admin/rooms/:roomId
```

**Parámetros de Ruta:**

| Parámetro | Descripción |
|---|---|
| `roomId` | UUID de la sala |

**Respuesta (202):**

```json
{
  "message": "Room deletion queued"
}
```

| Estado | Descripción |
|--------|-------------|
| 202 | Eliminación encolada — se completará en breve |
| 404 | Sala no encontrada |
| 500 | Error al obtener la sala o encolar la tarea |

> **Nota:** Al cerrar se envía un mensaje del sistema a todos los participantes activos: "This room has been closed by an administrator".
> La lógica de cierre llama a `cleanupSvc.CascadeDeleteRoom` que gestiona la eliminación de la sala en LiveKit, la limpieza de archivos subidos
> y la eliminación del registro en la base de datos.

---

### Suspender Sala

Desactivar una sala activa, finalizando todas las llamadas activas pero preservando el registro de la sala. La sala puede reactivarse cuando un participante se una nuevamente. Se ejecuta en segundo plano y regresa inmediatamente.

```
POST /api/admin/rooms/:roomId/suspend
```

**Parámetros de Ruta:**

| Parámetro | Descripción |
|---|---|
| `roomId` | UUID de la sala |

**Respuesta (202):**

```json
{
  "message": "Room suspend queued"
}
```

| Estado | Descripción |
|--------|-------------|
| 202 | Suspensión encolada — se completará en breve |
| 400 | Sala ya inactiva |
| 404 | Sala no encontrada |
| 500 | Error al obtener la sala o encolar la tarea |

---

### Suspender Varias Salas

Encolar la suspensión de varias salas a la vez. Las salas ya suspendidas se omiten.

```
POST /api/admin/rooms/bulk/suspend
```

**Cuerpo de la Solicitud:**

```json
{
  "ids": ["room-uuid-1", "room-uuid-2"]
}
```

**Respuesta (202):**

```json
{
  "results": {
    "room-uuid-1": {"success": true, "name": "team-standup"},
    "room-uuid-2": {"success": false, "name": "design-review", "error": "room not found"}
  },
  "totalProcessed": 1,
  "totalFailed": 1
}
```

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `results` | object | Resultado por sala, indexado por ID |
| `results.*.success` | bool | Si la operación fue encolada |
| `results.*.name` | string | Nombre de la sala (presente en éxito) |
| `results.*.error` | string | Mensaje de error (presente en fallo) |
| `totalProcessed` | int | Número de salas encoladas con éxito |
| `totalFailed` | int | Número de salas que fallaron |

### Cerrar Varias Salas

Encolar la eliminación permanente de varias salas a la vez. No se puede deshacer.

```
POST /api/admin/rooms/bulk/close
```

Misma forma de solicitud/respuesta que [Suspender Varias Salas](#suspender-varias-salas).

## Tokens de Invitación

Los tokens de invitación permiten un registro controlado. Cuando `tokenRegistrationOnly` está habilitado en la configuración, los nuevos usuarios deben proporcionar un token de invitación válido y no utilizado durante el registro.

### Listar Tokens de Invitación

Recupera todos los tokens de invitación, los más recientes primero. Cada token incluye un booleano `used` calculado a partir de `usedAt`.

```
GET /api/admin/invite-tokens
```

**Respuesta (200):**

```json
{
  "tokens": [
    {
      "id": "token-uuid-1",
      "token": "a1b2c3d4e5f6...",
      "email": "invited@example.com",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-06-15T10:30:00Z",
      "usedAt": null,
      "usedBy": "",
      "createdAt": "2025-03-15T10:30:00Z",
      "used": false
    },
    {
      "id": "token-uuid-2",
      "token": "f6e5d4c3b2a1...",
      "email": "",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-04-15T10:30:00Z",
      "usedAt": "2025-03-20T14:00:00Z",
      "usedBy": "new-user-uuid",
      "createdAt": "2025-03-12T10:30:00Z",
      "used": true
    }
  ]
}
```

---

### Crear Token de Invitación

Genera un nuevo token de invitación. El valor del token es una cadena hexadecimal aleatoria de 32 caracteres.

```
POST /api/admin/invite-tokens
```

**Cuerpo de la Solicitud:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| Campo | Tipo | Requerido | Valor predeterminado | Descripción |
|---|---|---|---|---|
| `email` | string | No | _ninguno_ | Restringir el token a esta dirección de correo |
| `expiresInHours` | int | No | 72 | Período de validez del token en horas |

**Respuesta (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

---

### Eliminar Token de Invitación

Revoca un token de invitación por su ID. Después de la eliminación, el token ya no puede usarse para el registro.

```

```

**Parámetros de Ruta:**

| Parámetro | Descripción |
|---|---|
| `id` | UUID del token de invitación |

**Respuesta (200):**

```json

}
```

### Información del Certificado

Obtiene los metadatos del certificado TLS incluyendo el estado de expiración.

```
GET /api/admin/cert-info
```

**Respuesta (200) — TLS habilitado, certificado válido:**

```json
{
  "enabled": true,
  "status": "valid",
  "daysRemaining": 78,
  "notAfter": "2025-07-30T12:00:00Z",
  "subject": "CN=bedrud.example.com",
  "issuer": "CN=R3,O=Let's Encrypt,C=US",
  "notBefore": "2025-04-01T12:00:00Z",
  "sans": ["bedrud.example.com", "www.bedrud.example.com"]
}
```

**Respuesta (200) — TLS no habilitado:**

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| Estado | Significado |
|---|---|
| `valid` | El certificado es válido y no está próximo a expirar |
| `expiring_soon` | El certificado expira en los próximos 30 días |
| `expired` | El certificado ha expirado |
| `not_configured` | TLS está deshabilitado |
| `error` | El archivo del certificado falta o no es válido |

## Respuestas de Error

```json
{
  "error": "forbidden"
}
```

| Estado | Significado |
|---|---|
| 400 | Entrada no válida (ej., JSON mal formado, campos requeridos faltantes, eliminar cuenta propia, suspender sala inactiva) |
| 401 | No autenticado — JWT faltante o expirado |
| 403 | No es superadmin — el usuario no tiene `superadmin` en su arreglo `accesses` |
| 404 | Recurso no encontrado (usuario, sala o token) |
| 409 | Conflicto — el usuario no puede eliminar su propia cuenta |
| 500 | Error interno del servidor |
| 501 | No implementado (ej., generación de token de sala para admin) |
| 503 | Información del certificado no disponible (error de TLS) |
