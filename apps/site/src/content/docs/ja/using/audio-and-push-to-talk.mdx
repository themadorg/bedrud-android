マイクとスピーカーを設定し、必要ならプッシュトゥトーク（PTT）を使い、キーを押している間だけ送信します。

## デバイス選択

**設定 → オーディオ**（およびプレ参加）で：

- ブラウザが権限を付与するとマイクとスピーカーの一覧が埋まるはずです。
- **通話中**のデバイス切り替えは、ルームの完全再接続なしで動作するべきです。

ビデオカメラは **設定 → ビデオ**。 [アカウントと設定](/ja/docs/using/account-and-settings) を参照。

## プッシュトゥトークを有効にする

1. **設定 → オーディオ** を開きます。
2. **プッシュトゥトーク**を有効にします。
3. キーキャプチャコントロールで**ホールドキー**を取得します。
4. UI が明示的な保存を要求する場合は保存 — 設定は再読み込み後も保持されるべきです。

## 会議中

| 手順 | 期待 |
|------|------|
| PTT キーを押したまま | マイクが**送信**；他者が聞こえる |
| キーを離す | マイクが**ミュート**；他者が聞こえなくなる |
| インジケーター | 実装時、ローカル操作とリモートタイルに PTT 状態が表示される場合あり |
| PTT を無効化 | 通常のミュートトグル動作に戻る |

静かにするつもりなのに PTT がオンのままマイクが開いたままにしないでください — その場合は PTT を無効にするか明示的にミュートします。

## デフ / ヘッドフォン

ビルドが**デフ**やヘッドフォン操作を出す場合、他者向けのリモート音声を固定させてはいけません。デフはローカルの聴取状態であり、他者のリモートミュートではありません（モデレーション操作と組み合わせる場合を除く）。
