ステージモードはルームレイアウトを切り替え、通常のグリッドやスポットライトではなく発表者（またはステージセット）を前面に出します。

## ステージを開始

1. ホスト（またはステージを制御できるロール）として、会議コントロールから**ステージを開始**します。
2. 他の参加者は**ステージレイアウト**と、ステージセッション向けの**参加通知**を見ます。
3. 参加者ストリップ / サイドバーにはルームの残りが表示される場合があります（トグルがあれば切り替え）。

## ステージ上の画面共有

- ステージ中に画面を共有 — 他者は**ステージ画面共有オーバーレイ**を見ます。
- **共有を停止**するとオーバーレイが消えます。

非ステージの画面共有（通常グリッド）は [ミーティングコントロール](/ja/docs/using/meeting-controls) を参照。

## ステージを終了

**ステージを終了**すると、ルームは通常のグリッドまたはスポットライトレイアウトに戻ります。

## 遅れて参加した人

ステージが**すでに有効な間に参加**した人は、正しいステージ状態をすぐに見るべきです — 空のグリッドのままではありません。遅れて参加した人が誤ったレイアウトを見る場合は一度再読み込みし、なお失敗する場合はリグレッションとして報告してください。

## ヒント

- ステージはデモ、教育、一対多のプレゼンに向いています。
- [YouTube ウォッチ](/ja/docs/using/youtube-watch) や [ホワイトボード](/ja/docs/using/whiteboard) と組み合わせるのは、それらのオーバーレイが共存する設計のときだけにし、参加者が複数の全画面レイヤーに閉じ込められないよう慎重に切り替えます。
