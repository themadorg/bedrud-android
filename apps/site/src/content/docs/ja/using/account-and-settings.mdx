アカウントを作成し、セッションを維持し、通話に参加する前にプロフィールとメディア設定を構成します。

## 登録とログイン

1. ブラウザで Bedrud インスタンスを開きます。
2. メールとパスワードで**登録**します（ホストが要求する招待トークンがあればそれも）。
3. メール確認が有効な場合、UI の指示に従って確認を完了します。
4. 正しい資格情報で**ログイン**します。誤ったパスワードは明確なエラーを表示し、セッションは作成されません。
5. アカウントメニューから**ログアウト**します。保護されたルート（ダッシュボード、設定、管理）は認証へリダイレクトされます。

アクセストークンが有効な間、セッションはページの**再読み込み**後も維持されます。

### その他のサインイン方法

インスタンスの設定に応じて、次も表示される場合があります。

- **OAuth**（Google、GitHub、Twitter、またはカスタム OIDC）
- **パスキー**（WebAuthn）— セキュリティ設定で登録後
- **ゲスト参加** — 許可されたルームではアカウント不要（[ミーティングへの参加](/ja/docs/using/joining-a-meeting) を参照）

パスワードを忘れた場合は、ログインページの**パスワードを忘れた**を使い、リセットリンクに従ってください（サーバーで SMTP が必要）。

運用者向けの詳細（JWT、パスキー、レート制限）は [認証](/ja/docs/backend/authentication) を参照してください。

## プロフィール

ダッシュボードから **設定 → プロフィール** を開きます。

- **名前** / 表示フィールドを編集して保存 — 再読み込み後も正しく再読み込みされます。
- 変更はダッシュボード、会議タイル、参加者リストに反映されます。

## アバター

引き続き **設定 → プロフィール** 内：

| ルール | 詳細 |
|--------|------|
| 形式 | JPEG、PNG、WebP |
| サイズ上限 | **2 MB** 未満 |
| 過大 / 不正な MIME | エラーで拒否（クラッシュしない） |
| 再アップロード | 以前のファイルを置き換え |

アップロード後、プレビューはすぐに更新されます。ダッシュボードの再読み込み後と会議内で新しいアバターを確認してください。

## 設定パネル

ダッシュボードの設定ルートから各パネルを開きます。

| パネル | 内容 |
|--------|------|
| **プロフィール** | 名前と表示 ID |
| **オーディオ** | マイク / スピーカー一覧；**プッシュトゥトーク**の有効化とホールドキー取得；再読み込み後も保存 |
| **ビデオ** | カメラデバイス選択；再読み込み後も保持 |
| **セキュリティ** | パスワード変更とパスキー登録 |
| **外観** | テーマと関連 UI トグル（表示時） |
| **実験的** | **共有ホワイトボード**や **YouTube ウォッチパーティー**などの任意機能 |

設定はアカウントに保存され、参加時に会議シェルへ同期される場合があります。通話中の PTT 動作は [オーディオとプッシュトゥトーク](/ja/docs/using/audio-and-push-to-talk) を参照してください。
