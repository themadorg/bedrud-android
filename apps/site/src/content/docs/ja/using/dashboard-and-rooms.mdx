ログイン後、ダッシュボードはルームの作成・参加、最近使ったルームへの再入場の拠点です。

## ルームを作成

1. ダッシュボードから新しいルームを作成します（名前とホストが公開するルームオプション）。
2. ルームがリストに表示されます。
3. ルームを開いてプレ参加 / 会議フローに入ります。

ルームカードは参加と削除をサポートします。レイアウト密度は UI バージョンで変わることがありますが、参加と削除は利用できます。

## クイック参加

**ルームスラッグ**または**完全な会議 URL** をクイック参加に貼り付けます。Bedrud がルームを解決し、正しい参加パスへ案内します。

共有リンクは通常次の形式です。

```
https://your-host/m/<room-id-or-slug>
```

## 最近のルーム

ルームに**参加**すると **最近のルーム** に追加されます。

- リストはページの**再読み込み後も残ります**。
- 最近のエントリを**削除** — 再読み込み後も削除されたままです。
- 最近のカードでそのルームを素早く開き直せます。

最近のルームはクライアント上の便利な一覧であり、所有またはアクセス可能なサーバー側ルーム一覧の代わりにはなりません。

## ランディングページ

インスタンスのルートを開くと：

- **ログアウト時** — マーケティング / サインインパス（壊れた API プロキシなし）
- **ログイン時** — ダッシュボード入口
