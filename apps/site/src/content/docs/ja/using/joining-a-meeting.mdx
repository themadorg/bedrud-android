Web アプリから Bedrud ルームに入る方法：プレ参加プレビュー、接続、基本メディア操作、アクセスゲート。

## プレ参加（ウェルカムシェル）

メディア接続前に通常表示されるもの：

- 表示**名**
- **マイク**と**カメラ**のプレビュー
- マイク、カメラ、スピーカーの**デバイス選択**（利用可能な場合）

参加前に設定し、最初の接続で正しいデバイスを使うようにします。

## 参加と接続

1. **参加**（または同等）をクリックします。
2. 短い読み込みシェルのあと、**接続済み**の会議 UI が表示されるはずです — 無限スピナーではありません。
3. 他の参加者は参加後にタイルまたはアバターとして表示されます。

接続に失敗した場合は、ブラウザの権限、ネットワーク、サーバーの LiveKit/WebRTC ポート到達性を確認してください。運用者：[WebRTC 接続性](/ja/docs/architecture/webrtc-connectivity) を参照。

## 基本操作

| 操作 | 期待される動作 |
|------|----------------|
| マイクのミュート / 解除 | 他の参加者があなたの声を聞こえなくなる / 聞こえるようになる |
| カメラ on / off | あなたのビデオタイルが他者向けに更新される |
| ルームを退出 | きれいに切断；他者がゾンビ参加者を永久に保持しない |
| 再参加 | 退出後、同じルームへの再参加が動作する |

## 複数参加者

2 つのブラウザ（または通常 + プライベート）で確認：

- 双方が互いのタイルを見る
- ミュートとカメラ状態がリモートで見える

## 制限付きルーム

ルームは**パスワード**、承認待ち、その他ルーム設定に応じたアクセス確認を要求する場合があります。表示されたら**ルームアクセスダイアログ**を完了し、プレ参加 / 参加へ進みます。

一部インスタンスではゲスト参加（アカウント不要）が許可されます — リンクを共有し、ゲストは表示名のみで入ります。ルームが満員、または定員トラッキングに失敗した場合、参加は失敗します（メディアトークンなし）。ゲストには**安定した署名付き ID Cookie**（`bedrud_guest_id`）が付き、ルーム BAN が再訪問でも維持されます。公開ルームのロビー**人数**は公開可；参加者の ID 一覧には認証が必要です。
