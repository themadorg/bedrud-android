管理者エンドポイントは `/api/admin/` 以下にあります。すべてのエンドポイントは **superadmin** アクセスレベルと、有効なJWTを `Authorization: Bearer <token>` ヘッダーに含める必要があります。

## 概要

| エンドポイント | メソッド | 説明 |
|---|---|---|
| `/api/admin/users` | GET | 全ユーザーを一覧表示（ページネーション対応） |
| `/api/admin/users/:id` | GET | ユーザー詳細とルーム情報を取得 |
| `/api/admin/users/:id/status` | PUT | ユーザーを有効化または無効化 |
| `/api/admin/users/:id/accesses` | PUT | ユーザーのアクセスレベルを更新 |
| `/api/admin/users/:id` | DELETE | ユーザーを完全に削除 |
| `/api/admin/rooms` | GET | 全ルームを一覧表示（ページネーション対応） |
| `/api/admin/rooms/:roomId` | PUT | 任意のルーム設定を更新 |
| `/api/admin/rooms/:roomId` | DELETE | ルームを閉鎖（カスケード削除） |
| `/api/admin/rooms/:roomId/suspend` | POST | アクティブなルームを一時停止 |
| `/api/admin/rooms/:roomId/participants` | GET | LiveKitからライブ参加者データを取得 |
| `/api/admin/rooms/:roomId/participants/:identity/kick` | POST | 任意の参加者をルームからキック |
| `/api/admin/rooms/:roomId/participants/:identity/mute` | POST | 任意の参加者の音声をミュート |
| `/api/admin/settings` | GET | システム設定を取得（シークレットはマスク） |
| `/api/admin/settings` | PUT | システム設定を更新 |
| `/api/admin/invite-tokens` | GET | 全招待トークンを一覧表示 |
| `/api/admin/invite-tokens` | POST | 新しい招待トークンを作成 |
| `/api/admin/invite-tokens/:id` | DELETE | 招待トークンを削除 |
| `/api/admin/online-count` | GET | 全アクティブ参加者数 |
| `/api/admin/livekit/stats` | GET | LiveKitサーバー統計の集計 |
| `/api/admin/cert-info` | GET | TLS証明書メタデータ |

## アクセスレベル

Bedrudは複数レベルのアクセスシステムを採用しています。各ユーザーは `accesses` 配列を持ち、複数のレベルを同時に保持できます。

| アクセスレベル | 説明 | ミドルウェア保護 |
|---|---|---|
| `superadmin` | システム全体へのフルアクセス。管理パネル、ユーザー/ルーム管理、設定、招待トークン。 | `RequireAccess("superadmin")` （`/api/admin/*` に適用） |
| `admin` | 定義済みだがミドルウェアで強制されていない。情報提供のみ。 | 現在なし |
| `moderator` | ルーム単位のモデレーション。チャットブロック、ミュート解除/ミュート、スポットライト、ルームごとのビデオ無効化。 | ルームスコープの `isRoomModerator()` チェック |
| `user` | 標準の認証済みユーザー。ルームの作成・参加、プロフィール管理。 | ユーザールートに `Protected()` を適用 |
| `guest` | 公開ルーム参加用の一時ユーザー。ゲストログインのみ。 | 該当なし |

> **注意:** `/api/admin/*` ルートグループは `admin` ではなく `superadmin` をチェックします。`accesses` 配列に `superadmin` を含むユーザーのみがこれらのエンドポイントを使用できます。

## ユーザー管理

### 全ユーザー一覧の取得

登録済みの全ユーザーをページネーション付きで取得します。

```
GET /api/admin/users?page=1&limit=50
```

**ヘッダー:** `Authorization: Bearer <accessToken>`

**クエリパラメータ:**

| パラメータ | 型 | デフォルト | 説明 |
|---|---|---|---|
| `page` | int | 1 | ページ番号 |
| `limit` | int | 50 | 1ページあたりのユーザー数 |

**レスポンス (200):**

```json
{
  "users": [
    {
      "id": "123e4567-e89b-12d3-a456-426614174000",
      "email": "user@example.com",
      "name": "John Doe",
      "provider": "local",
      "isActive": true,
      "isAdmin": false,
      "accesses": ["user"],
      "createdAt": "2025-01-15 10:30:00"
    },
    {
      "id": "123e4567-e89b-12d3-a456-426614174001",
      "email": "admin@example.com",
      "name": "Admin User",
      "provider": "local",
      "isActive": true,
      "isAdmin": true,
      "accesses": ["user", "superadmin"],
      "createdAt": "2025-01-10 08:00:00"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `id` | ユーザーUUID |

**レスポンス (200):**

```json
{
  "user": {
    "id": "123e4567-e89b-12d3-a456-426614174000",
    "email": "user@example.com",
    "name": "John Doe",
    "provider": "local",
    "isActive": true,
    "isAdmin": false,
    "accesses": ["user"],
    "createdAt": "2025-01-15 10:30:00"
  },
  "rooms": [
    {
      "id": "room-uuid-1",
      "name": "team-standup",
      "createdAt": "2025-02-01T09:00:00Z"
    }
  ]
}
```

### ユーザーステータスの更新

ユーザーアカウントを有効化または無効化します。無効化されたユーザーはログインやトークンの更新ができません。

```
PUT /api/admin/users/:id/status
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `id` | ユーザーUUID |

**リクエストボディ:**

```json
{
  "active": false
}
```

**レスポンス (200):**

```json
{
  "message": "User status updated successfully"
}
```

---

### ユーザーアクセス権限の更新

ユーザーのアクセスレベル配列を変更します。superadminへの昇格、降格、またはmoderatorアクセスの追加に使用します。

```
PUT /api/admin/users/:id/accesses
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `id` | ユーザーUUID |

**リクエストボディ:**

```json
{
  "accesses": ["user", "superadmin"]
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| `accesses` | string[] | アクセスレベルの配列。有効な値: `user`, `superadmin`, `admin`, `moderator`, `guest` |

**レスポンス (200):**

```json
{
  "message": "User accesses updated"
}
```

---

### ユーザーの削除

ユーザーと関連するすべてのデータを完全に削除します。非同期で実行され、即座に `202 Accepted` を返します。自分のアカウントは削除できません。

```
DELETE /api/admin/users/:id
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `id` | ユーザーUUID |

**レスポンス (202):**

```json
{
  "message": "User deletion queued",
  "rooms": 3
}
```

| フィールド | 説明 |
|---|---|
| `message` | 削除が進行中であることの確認 |
| `rooms` | このユーザーが作成したルーム数（これらも削除されます） |

バックグラウンドジョブは以下をカスケード削除します:
- ユーザーが作成したすべてのルーム
- 関連するパスキー（WebAuthn認証情報）
- ユーザー設定
- ユーザーレコード自体

## ルーム管理

### 全ルーム一覧の取得

インスタンス上の全ルームをページネーション付きで取得します（作成者フィルターをバイパスします）。

```
GET /api/admin/rooms?page=1&limit=50
```

**クエリパラメータ:**

| パラメータ | 型 | デフォルト | 説明 |
|---|---|---|---|
| `page` | int | 1 | ページ番号 |
| `limit` | int | 50 | 1ページあたりのルーム数 |

**レスポンス (200):**

```json
{
  "rooms": [
    {
      "id": "room-uuid-1",
      "name": "team-standup",
      "createdBy": "user-uuid",
      "maxParticipants": 25,
      "isActive": true,
      "settings": {
        "allowChat": true,
        "allowAudio": true,
        "allowVideo": true,
        "isPersistent": false
      },
      "createdAt": "2025-03-01T10:00:00Z"
    }
  ],
  "total": 8,
  "page": 1,
  "limit": 50
}
```

---

### ルームの更新

管理者専用の `isPersistent` フラグを含む、任意のルーム設定を変更します。

```
PUT /api/admin/rooms/:roomId
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `roomId` | ルームUUID |

**リクエストボディ:**

```json
{
  "maxParticipants": 50,
  "settings": {
    "allowChat": true,
    "allowAudio": true,
    "allowVideo": true,
    "requireApproval": false,
    "e2ee": false,
    "isPersistent": true
  }
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| `maxParticipants` | int | 許可する最大参加者数（任意） |
| `settings.allowChat` | bool | ルーム内チャットを有効にする |
| `settings.allowAudio` | bool | オーディオトラックを有効にする |
| `settings.allowVideo` | bool | ビデオトラックを有効にする |
| `settings.requireApproval` | bool | 参加に管理者の承認を必要とする |
| `settings.e2ee` | bool | エンドツーエンド暗号化を有効にする |
| `settings.isPersistent` | bool | **管理者専用。** アイドル時の自動クリーンアップを防止 |

すべての設定フィールドは任意です。変更したいフィールドのみを含めてください。

**レスポンス (200):** 更新されたルームオブジェクトを返します。

---

### ルームの閉鎖

カスケード削除でルームを完全に閉鎖します。全参加者にシステムメッセージ「このルームは管理者によって閉鎖されました」を送信します。

```
DELETE /api/admin/rooms/:roomId
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `roomId` | ルームUUID |

**レスポンス (200):**

```json
{
  "status": "success"
}
```

---

### ルームの一時停止

ルームレコードを削除せずに、アクティブなルームとその参加者全員を非アクティブにします。参加者が再度参加するとルームは再アクティブ化されます。

```
POST /api/admin/rooms/:roomId/suspend
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `roomId` | ルームUUID |

**レスポンス (200):**

```json
{
  "status": "success"
}
```

**エラー (400):** ルームが既に非アクティブな場合、`{"error": "Room is not active"}` を返します。

### 参加者のキック

任意の参加者をルームから削除します（作成者所有権チェックは不要です）。

```

```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `roomId` | ルームUUID |
| `identity` | 参加者ID（ユーザーUUID） |

**レスポンス (200):**

```json
{
  "status": "success"
}
```

---

### 参加者のミュート

ルーム内の任意の参加者の全オーディオトラックをミュートします。

```
POST /api/admin/rooms/:roomId/participants/:identity/mute
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `roomId` | ルームUUID |
| `identity` | 参加者ID（ユーザーUUID） |

**リクエストボディ:**

```json
{
  "muted": true
}
```

**レスポンス (200):**

```json
{
  "status": "success"
}
```

### 設定の更新

システム設定を更新します。シークレットフィールドは `"••••••••"` または空文字列を送信することで既存の値を保持できます。サーバーは実際の値が送信された場合のみシークレットを上書きします。

```
PUT /api/admin/settings
```

**リクエストボディ（部分的な例）:**

```json
{
  "registrationEnabled": false,
  "tokenRegistrationOnly": true,
  "passkeysEnabled": true,
  "googleClientId": "your-client-id.apps.googleusercontent.com",
  "googleClientSecret": "GOCSPX-real-secret-value",
  "googleRedirectUrl": "https://your-server.com/api/auth/google/callback",

  "logLevel": "debug"
}
```

## 招待トークン

招待トークンは制限付き登録を有効にします。設定で `tokenRegistrationOnly` が有効な場合、新規ユーザーは登録時に有効かつ未使用の招待トークンを提供する必要があります。

### 招待トークン一覧の取得

全招待トークンを新しい順に取得します。各トークンには `usedAt` から計算された `used` 真偽値が含まれます。

```
GET /api/admin/invite-tokens
```

**レスポンス (200):**

```json
{
  "tokens": [
    {
      "id": "token-uuid-1",
      "token": "a1b2c3d4e5f6...",
      "email": "invited@example.com",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-06-15T10:30:00Z",
      "usedAt": null,
      "usedBy": "",
      "createdAt": "2025-03-15T10:30:00Z",
      "used": false
    },
    {
      "id": "token-uuid-2",
      "token": "f6e5d4c3b2a1...",
      "email": "",
      "createdBy": "admin-uuid",
      "expiresAt": "2025-04-15T10:30:00Z",
      "usedAt": "2025-03-20T14:00:00Z",
      "usedBy": "new-user-uuid",
      "createdAt": "2025-03-12T10:30:00Z",
      "used": true
    }
  ]
}
```

---

### 招待トークンの作成

新しい招待トークンを生成します。トークン値は32文字のランダムな16進数文字列です。

```
POST /api/admin/invite-tokens
```

**リクエストボディ:**

```json
{
  "email": "invited@example.com",
  "expiresInHours": 168
}
```

| フィールド | 型 | 必須 | デフォルト | 説明 |
|---|---|---|---|---|
| `email` | string | いいえ | _なし_ | このメールアドレスにトークンを制限 |
| `expiresInHours` | int | いいえ | 72 | トークンの有効期間（時間） |

**レスポンス (201):**

```json
{
  "id": "uuid-string",
  "token": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6",
  "email": "invited@example.com",
  "createdBy": "admin-uuid",
  "expiresAt": "2025-03-22T10:30:00Z",
  "usedAt": null,
  "usedBy": "",
  "createdAt": "2025-03-15T10:30:00Z"
}
```

---

### 招待トークンの削除

IDで招待トークンを無効化します。削除後、そのトークンは登録に使用できなくなります。

```
DELETE /api/admin/invite-tokens/:id
```

**パスパラメータ:**

| パラメータ | 説明 |
|---|---|
| `id` | 招待トークンUUID |

**レスポンス (200):**

```json
{
  "status": "success"
}
```

### 証明書情報

有効期限の状態を含むTLS証明書のメタデータを取得します。

```
GET /api/admin/cert-info
```

**レスポンス (200) — TLS有効、証明書有効:**

```json
{
  "enabled": true,
  "status": "valid",
  "daysRemaining": 78,
  "notAfter": "2025-07-30T12:00:00Z",
  "subject": "CN=bedrud.example.com",
  "issuer": "CN=R3,O=Let's Encrypt,C=US",
  "notBefore": "2025-04-01T12:00:00Z",
  "sans": ["bedrud.example.com", "www.bedrud.example.com"]
}
```

**レスポンス (200) — TLS無効:**

```json
{
  "enabled": false,
  "status": "not_configured"
}
```

| ステータス | 意味 |
|---|---|
| `valid` | 証明書は有効で、期限切れが近くありません |
| `expiring_soon` | 証明書の有効期限が30日以内です |
| `expired` | 証明書の有効期限が切れています |
| `not_configured` | TLSが無効です |
| `error` | 証明書ファイルが見つからないか無効です |

---

## エラーレスポンス

```json
{
  "error": "forbidden"
}
```

| ステータス | 意味 |
|---|---|
| 400 | 不正な入力（例: 不正なJSON、必須フィールドの欠落、自分のアカウントの削除、非アクティブルームの一時停止） |
| 401 | 未認証 — JWTが欠落または期限切れ |
| 403 | superadminではありません — ユーザーの `accesses` 配列に `superadmin` が含まれていません |
| 404 | リソースが見つかりません（ユーザー、ルーム、またはトークン） |
| 409 | 競合 — ユーザーは自分のアカウントを削除できません |
| 500 | サーバー内部エラー |
| 501 | 未実装（例: 管理者ルームトークン生成） |
| 503 | 証明書情報が利用できません（TLSエラー） |
