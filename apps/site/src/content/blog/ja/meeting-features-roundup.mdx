セルフホストのビデオは「参加して顔を見る」だけではありません。Bedrud の会議シェルは、プレゼンレイアウト、より豊かなチャット、任意のウォッチパーティー、プッシュトゥトークまでカバーします — それでもあなたのハードウェア上の 1 バイナリです。

この記事は地図です。深いハウツーは docs の **Bedrud の使い方** にあります。

## ステージモード

フリーフォーオールのグリッドと戦わずにプレゼン。ホストは**ステージ**を開始し、ステージオーバーレイとして画面を共有し、参加者ストリップを保ち、ステージを終了して通常レイアウトに戻れます。遅れて参加した人は正しいステージ状態に着地するべきです。

→ [ステージモード](/ja/docs/using/stage-mode)

## 共同ホワイトボード

LiveKit データチャネル上の Yjs による共有描画。マルチユーザーのストローク、選択/移動/削除、任意の実験的トグル。

→ [共同ホワイトボード](/ja/docs/using/whiteboard) · [エンジニアリング記事](/ja/blog/collaborative-whiteboard)

## YouTube ウォッチ

URL を貼り付け、おおよそ同期して再生し、全員向けに閉じる。ゲート時は実験的設定の後ろでも利用可能。

→ [YouTube ウォッチ](/ja/docs/using/youtube-watch)

## チャット：リアクション、アンケート、添付

ルーム内テキストはスタブではありません：

- 絵文字**リアクション**とコンポーザーの**絵文字ピッカー**
- ライブ投票更新付きの**アンケート**
- 枠内の**画像 / ファイル**（大きなペイロードはチャンク）
- 長いセッションを有界に保つクライアント側の保持上限

→ [会議内チャット](/ja/docs/using/chat)

## プッシュトゥトーク

オーディオ設定で PTT を有効にし、ホールドキーを取得し、押している間だけ送信。無効化で通常のミュートトグルに戻す。通話中のデバイス切り替えは完全再接続なし。

→ [オーディオとプッシュトゥトーク](/ja/docs/using/audio-and-push-to-talk)

## ダッシュボードとアカウントの基本

通話前：登録、アバター、最近のルーム、スラッグや URL でのクイック参加。

→ [アカウントと設定](/ja/docs/using/account-and-settings) · [ダッシュボードとルーム](/ja/docs/using/dashboard-and-rooms) · [ミーティングへの参加](/ja/docs/using/joining-a-meeting)

## モデレーションとレイアウト

ステージ外の画面共有、スポットライト、ルーム情報、ロールベースのコンテキストメニュー、敵対的ネットワーク時のトランスポートフォールバック UX。

→ [ミーティングコントロール](/ja/docs/using/meeting-controls) · [ユーザーロール](/ja/docs/guides/roles)

## まだ計画中のもの

**録画**ルートは計画中 / 運用者向けの経路のまま — docs がゲートを外すまで、完全な egress 録画をすべてのリリースで本番完備と見なさないでください。インフラ、管理、LiveKit、デプロイガイドが運用者の真実の源です。

## 機能ページ

これらの会議能力のマーケティングカードは [機能](/ja/features) にあり、それぞれ上記 docs へリンクします。
