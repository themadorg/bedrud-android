# Bedrud Server Documentation

The Bedrud server is a single-binary Go backend that powers the video meeting platform. It embeds the React web frontend and LiveKit media server, exposes a REST API over Fiber, and ships a full CLI for installation and administration.

**Go module:** `bedrud`  
**Go version:** 1.26  
**Root directory:** `server/`

**Full monorepo tree:** [docs/project-tree.md](../project-tree.md)

**Public docs (bedrud.org):** [Backend Documentation](https://bedrud.org/en/docs/backend/overview) вҖ” operator-facing guides, deployment, config reference. Cross-reference: [public-docs.md](./public-docs.md).

### Key Technologies

| Layer | Technology |
|-------|------------|
| HTTP framework | [Fiber v2](https://gofiber.io/) |
| ORM | [GORM](https://gorm.io/) вҖ” SQLite (dev) / PostgreSQL (prod) |
| Media | [LiveKit](https://livekit.io/) вҖ” embedded or external |
| Auth | JWT, OAuth2 (Goth), WebAuthn passkeys |
| Logging | [Zerolog](https://github.com/rs/zerolog) |
| CLI | [Cobra](https://github.com/spf13/cobra) + [Viper](https://github.com/spf13/viper) |
| Scheduling | [gocron](https://github.com/go-co-op/gocron) |
| API docs | Swagger / Scalar (generated by `swag`) |

### Layered Design

1. **Server layer** (`internal/server`) вҖ” Fiber app, routes, middleware, TLS, SPA serving
2. **Handler layer** (`internal/handlers`) вҖ” HTTP request/response logic
3. **Repository layer** (`internal/repository`) вҖ” GORM data access
4. **Model layer** (`internal/models`) вҖ” Database schemas and domain types
5. **Queue layer** (`internal/queue`) вҖ” Async job worker (delete, suspend, email, webhooks)
6. **Service layer** (`internal/services`) вҖ” Cross-cutting business logic (cleanup, recordings)
7. **Specialized packages** вҖ” `auth`, `livekit`, `storage`, `middleware`, `scheduler`, `install`

## Quick Start

```bash
# From repo root
make init          # LiveKit binary + config + deps
make dev           # Hot-reload server + frontend

# Or server only
cd server
cp config.local.yaml.example config.yaml   # edit secrets
go run ./cmd/bedrud run
```

**Health checks:** `GET /api/health`, `GET /api/ready`  
**Swagger UI:** `http://localhost:8090/api/swagger`  
**Scalar UI:** `http://localhost:8090/api/scalar`
